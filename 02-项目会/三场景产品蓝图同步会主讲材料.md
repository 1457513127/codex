# 三场景私域变现产品蓝图同步会主讲材料

版本：V1.0

日期：2026-05-27

会议性质：产品方向同步会与前置任务启动会

主讲人：产品负责人

权威规划：[三场景私域变现平台产品蓝图](../01-研究与规划/三场景私域变现平台产品蓝图.md)

## 1. 会议主旨

本次会议不是现场收集大家想做什么，也不是请团队投票是否进入该方向。产品侧需要向团队说明：

1. 平台要服务哪类经营者，以及为什么从私域变现切入。
2. 三类首轮交易场景分别做什么、未来扩什么。
3. 研发现在应预设计和预开发哪些共用能力。
4. 运营现在应摸清哪些人群、供给和真实业务规则。
5. 哪些风险必须在真实上线前完成支付、履约或合规核验。

## 2. 一句话产品方向

> 为经营型团长提供可开店、可发布供给、可一级帮卖、可追踪订单与收益的私域变现平台，首轮覆盖活动、电商商品和付费内容/课程包。

会议中应明确：

- 产品版图已确定为三场景，开发可以据此做模块与底座预研。
- 真正上线仍分期，未完成核验的资金或履约能力不能先承诺。
- 不做多级分销；本地服务、广告和直播训练营不在本轮范围。

## 3. 本次需要同步的产品版图

| 产品层 | 功能模块 | 团队理解重点 |
| --- | --- | --- |
| 经营入口 | 团长账号、店铺主页、订阅套餐 | 团长是主要经营用户和订阅付费方 |
| 供给能力 | 活动、商品、知识内容发布与审核 | 团长既能发布，也能推广他人供给 |
| 分发成交 | 一级帮卖、终端定价、分享归因、订单 | 推广可以加价，但不能发展下级 |
| 收益结算 | 订阅、交易抽佣、发布方结算、推广收益、退款冲回 | 平台费按终端成交额计算 |
| 履约售后 | 核销、物流、内容授权、退款与争议 | 三类供给有不同交付责任 |
| 运营治理 | 审核、暂停、工单、看板、风险清单 | 真实开放前要有治理手段 |

## 4. 场景讲解顺序

### 4.1 活动

- 团长发布活动或帮卖允许分发的活动。
- 消费者报名支付、参加活动、核销或退款。
- 开发需预研场次、名额、核销、活动取消与退款的状态链路。
- 运营需梳理首批可供给活动、退改规则和履约责任。

### 4.2 电商商品

- 发布团长/商家提供商品、设置底价并承担发货和商品售后。
- 推广团长可加价分享，但不承担发货履约。
- 开发需预研规格、可售/库存、物流、退换及收益回退。
- 运营需摸清供给来源、退换承诺、质量与售后处理现实成本。

### 4.3 付费内容/课程包

- 发布团长上传或配置付费内容/课程包入口。
- 消费者支付后获得受控访问权。
- 开发需预研授权发放、失效/退款和订单交付状态。
- 运营需核实内容版权、交付承诺、团长愿意售卖的内容类型。

## 5. 定价和收益如何解释

| 事项 | 会议统一说法 |
| --- | --- |
| 订阅 | 经营型团长为店铺和经营工具支付套餐费 |
| 成交抽佣 | 平台按消费者支付的终端成交额收取交易服务费 |
| 团长加价 | 一级推广团长可以在底价与平台费约束之上设置终端售价 |
| 发布方收入 | 帮卖成交时按其设置的供给结算底价获得应结算金额 |
| 推广方收入 | 获得终端售价扣除底价和平台服务费后的收益 |
| 退款 | 全退全回退，部分退款按比例冲回对应收益 |
| 自动结算 | 作为目标能力规划，必须先核验现有运行结算链路可复用性 |

## 6. 开发收到的预研任务

| 优先级 | 预研项 | 交付物 |
| --- | --- | --- |
| P0 | 团长双角色、店铺和订阅权限模型 | 角色/权限矩阵与依赖 |
| P0 | 三场景共用 `Offering` 与扩展字段 | 数据对象草案和场景差异表 |
| P0 | 一级帮卖、可加价定价与归因链路 | 定价校验、归因规则和接口草案 |
| P0 | 订单、交付、售后与结算账本 | 各场景状态机及退款回退处理 |
| P0 | 现有自动结算链路复用 | 可复用能力、缺口、支付/合规前置 |
| P1 | 运营审核、暂停供给和数据看板 | 最小后台能力与导出字段 |

开发当前不是承诺立即上线所有能力，而是为三场景分期开放给出工程可行性、共用底座与模块拆解。

## 7. 运营收到的市场摸排任务

| 场景 | 本轮摸排对象 | 需要带回的证据 |
| --- | --- | --- |
| 活动 | 现有活动方、可发布活动的团长 | 供给数量、价格、退改、履约风险、帮卖意愿 |
| 商品 | 有稳定商品来源的团长/商家 | 品类、底价、发货、退换、质量责任、加价接受度 |
| 知识内容 | 有课程或内容资产的团长/主理人 | 授权、交付方式、退款规则、推广意愿 |
| 团长侧 | 经营型团长 | 订阅付费意愿、三类选卖偏好、定价和收益预期 |

## 8. 支付与合规核验任务

| 核验事项 | 输出要求 |
| --- | --- |
| 已有结算链路 | 是否可支持发布者与推广者双角色以及平台按终端额抽佣 |
| 退款冲回 | 是否可实现全退和部分退款按比例冲回 |
| 商品责任 | 发布方发货/售后、平台争议入口的协议及披露要求 |
| 内容授权 | 付费内容版权、访问授权和退款条件 |
| 订阅收费 | 团长套餐、续费/取消、与交易收费并存的展示及协议 |

## 9. 建议会议议程

| 时间 | 内容 | 主讲/输入 | 会后结果 |
| ---: | --- | --- | --- |
| 0-10 分钟 | 项目定位与当前业务基础 | 产品 | 团队理解方向和历史依据 |
| 10-25 分钟 | 三场景功能蓝图 | 产品 | 明确活动、商品、内容模块 |
| 25-40 分钟 | 通用底座、定价与结算规则 | 产品 | 明确预开发共性 |
| 40-55 分钟 | 可点击原型走查 | 产品 | 形成界面与模块共识 |
| 55-70 分钟 | 开发预研任务同步 | 产品/研发 | 技术预研清单和反馈节点 |
| 70-82 分钟 | 运营摸排与供给任务同步 | 产品/运营 | 市场摸排清单和交付日期 |
| 82-90 分钟 | 支付合规核验及下一次汇报 | 产品/相关负责人 | 负责人、截止日、下次同步输入 |

## 10. 会后产物

- 开发：三场景数据对象、模块拆分、结算复用与排期风险初稿。
- 运营：三类供给和经营型团长摸排表，标注可发布/可帮卖及责任风险。
- 支付/合规：自动结算、退款冲回、订阅与三场景责任核验结论。
- 产品：根据反馈迭代蓝图和原型，不把未通过可行性核验的能力写成已上线功能。

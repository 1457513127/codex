版本：V2.0

会议性质：产品方向同步会与前置任务启动会

权威规划：[社群店铺驱动的三场景私域变现平台产品蓝图](../01-研究与规划/三场景私域变现平台产品蓝图.md)

## 1. 会议要让团队带走什么

本次会议不是现场堆功能，也不是重新讨论是否做私域变现。产品侧要同步清楚一条可执行主线：

> 团长注册社群并选择类型，平台生成专属店铺模板；团长从审核货盘发布或选卖活动、商品与本地服务，借助分享和优惠促成交易；平台记录履约、售后与收益，并在支付规则通过核验后结算。

会后各团队应明确：

1. 开发预开发的通用底座和三类履约差异。
2. 运营应建立的社群模板、供给白名单与审核材料。
3. 支付/合规必须先验证的结算、退款与责任规则。
4. AI 营销、群接龙和多级分销不是首发建设任务。

## 2. 这次修正了什么

| 之前表述 | 当前口径 |
| --- | --- |
| 活动、电商、付费内容/课程包三场景 | 活动、电商、本地服务三场景 |
| 本地服务暂不纳入 | 仅开放到店预约核销型低风险本地服务 |
| 内容作为独立模块 | 电子券/次卡归入电商虚拟权益；复杂内容后置 |
| 团长店铺是货架入口 | 店铺由社群主类型和兴趣标签匹配展示模板 |
| 帮卖以加价为主 | 首发同时支持固定佣金分享赚与加价帮卖，两者互斥且均为一级 |

## 3. 产品版图

| 产品层 | 首发建设内容 | 边界 |
| --- | --- | --- |
| 社群经营入口 | 团长账号、社群主类型/标签、专属店铺模板、订阅 | 同一平台小程序内生成主页，不做独立小程序 |
| 审核货盘 | 发布申请、选卖、审核、上下架、白名单 | 不允许未审核自由发布成交 |
| 三场景交易 | 活动票务、电商三种交付、本地到店服务 | 不进入高责任品类或上门服务 |
| 分发促销 | 专属分享归因、优惠券、固定佣金或加价帮卖 | 订单仅一名直接推广方，无多级 |
| 交易治理 | 多履约订单、核销/发货/权益发放、售后、收益账本 | 自动结算需先核验 |

## 4. 场景讲解

### 4.1 活动变现

- 覆盖亲子活动、青年兴趣活动和创业交流活动等标准化票务/体验。
- 首发流程为场次与票种上架、消费者报名支付、现场核销、取消/退款处理。
- 运营先建立低风险活动白名单，不开放高风险户外、复杂行程或强效果承诺活动。

### 4.2 电商变现

- 首发同时支持物流商品、本地自提商品、电子券/次卡等虚拟权益商品。
- 发布方承担商品资料、交付和售后责任；推广团长承担推荐关系披露与其发放的优惠成本。
- 开发需将 `logistics / pickup / virtual_voucher` 作为不同履约类型建模，而不是混成一个发货状态。

### 4.3 本地服务变现

- 首发仅支持到店预约核销服务，例如摄影、洗护、运动体验或亲子配套服务。
- 服务详情需披露门店、预约时段、服务规格、取消规则、履约方与投诉入口。
- 不开放上门维修、保洁、医疗美容或强效果承诺服务。

## 5. 收益与营销能力

| 事项 | 会议统一口径 |
| --- | --- |
| 平台收入 | 团长工具订阅费 + 按消费者实付额计提的交易服务费 |
| 固定佣金分享赚 | 发布方设佣金，推广团长分享成交后获取固定收益 |
| 加价帮卖 | 发布方设底价，推广团长设终端售价并获得扣除平台费后的价差收益 |
| 模式互斥 | 单个供给只能选择固定佣金或加价帮卖，一单只有一个直接推广方 |
| 优惠券 | 首发支持店铺券/单品券；推广方发券优先从推广收益承担且不得产生负收益 |
| 退款冲回 | 全额退款全链回退；部分退款按比例冲回收益与优惠分摊 |
| 自动结算 | 作为目标能力进入接口设计，不在支付链路核验前承诺真实打款 |

## 6. 首发与二期的分界

| 首发必须完成 | 提前规划但二期再做 |
| --- | --- |
| 社群类型与店铺模板 | AI 多渠道营销话术草稿 |
| 审核货盘和三类交易入口 | 群接龙/凑单流程 |
| 分享归因、固定佣金、加价帮卖 | 自动选货与自动上架 |
| 优惠券、订单、履约、售后、收益账本 | 外部平台自动发布 |
| 运营审核、白名单、风险暂停 | 多级分销或广告变现 |

## 8. 运营与合规任务

| 责任方 | 需要带回的材料 |
| --- | --- |
| 运营 - 社群 | 主类型与兴趣标签候选、各类型团长名单、模板货架建议 |
| 运营 - 活动 | 首发票务/体验供给、退改规则、履约风险和推广模式 |
| 运营 - 商品 | 物流/自提/虚拟权益白名单、交付证明、退换责任与库存规则 |
| 运营 - 服务 | 到店预约服务名单、服务方资质、取消与投诉处理承诺 |
| 支付/财务 | 收费主体、平台费、优惠承担、结算及退款回退可执行结论 |
| 合规/负责人 | 平台规则、主体披露、限制品类、消费者争议处理与营销审核边界 |

## 9. 建议议程

| 时间 | 同步内容 | 会后结果 |
| ---: | --- | --- |
| 0-10 分钟 | 产品主线与口径修正 | 理解社群店铺驱动定位 |
| 10-25 分钟 | 三场景白名单与履约方式 | 明确做什么与不做什么 |
| 25-40 分钟 | 店铺模板、货盘与推广/优惠规则 | 明确共用底座 |
| 40-55 分钟 | 可点击原型走查 | 对齐页面与操作链 |
| 55-70 分钟 | 开发与支付预研任务 | 确定技术结论交付 |
| 70-82 分钟 | 运营供给与社群摸排任务 | 确定白名单与负责人 |
| 82-90 分钟 | 风险、截止日和下一次同步输入 | 形成任务清单 |

## 10. 会议结论表达

- 三场景均进入受控真实首发规划，但仅对审核通过的低风险供给开放。
- 首发核心是社群建店、货盘成交、履约售后和一级收益闭环；AI 与群接龙不得抢占底座开发。
- 本轮交付仍为规划与可点击原型；真实支付、结算和品类开放以各责任方核验结果为门槛。

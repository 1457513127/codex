# 和趣项目首次全员介绍版主讲材料

版本：V3.0

日期：2026-05-27

会议性质：产品立项会 / 第一次全员了解项目

权威规划：[和趣社群团购与本地交易变现平台产品蓝图](../01-研究与规划/三场景私域变现平台产品蓝图.md)

## 1. 开场要讲清的一句话

和趣要做的是一个面向团长、群主和私域经营者的社群团购与本地交易变现平台。

它不是单点活动报名工具，也不是普通小程序商城，而是用“商品库 + 开团发布 + 分销帮卖 + 订单履约 + 售后结算”把社群里的交易变成可管理、可追踪、可结算的经营闭环。

## 2. 我们在哪个赛道

| 层级 | 赛道 | 和趣的进入方式 |
| --- | --- | --- |
| 主赛道 | 私域电商 / 社群团购 | 团长和群主通过微信群、小程序和熟人信任完成交易 |
| 延展赛道 | 本地生活交易 | 活动报名、卡券核销、到店预约服务都属于本地履约交易 |
| 首期切口 | 活动、普通团购、预售团购、卡券团购、到店服务 | 用低风险供给验证交易闭环，不一开始做全品类平台 |

现场讲法：我们不是和美团、抖音、淘宝正面抢公域流量，而是服务已经有社群触达的人，让他们把社群信任转成可履约、可售后、可分钱的交易。

## 3. 为什么现在值得做

| 信号 | 对和趣的意义 |
| --- | --- |
| 公域获客贵，私域反复触达价值提升 | 团长/群主需要低成本承接和转化工具 |
| 微信小程序已经是高频交易入口 | 社群分享到小程序交易链路更自然 |
| 私域电商从粗放卖货走向精细化经营 | 只靠群接龙和人工统计越来越难支撑售后和结算 |
| 本地活动、卡券和服务需要履约闭环 | 线下交易比纯商品更依赖核销、退款、责任披露 |

数据口径：

- 艾瑞咨询《2025年中国私域电商行业趋势白皮书》显示，私域电商 2024 年约 2.3 万亿元，2025 年预计突破 3 万亿元，2027 年用户规模预计 9.4 亿。
- QuestMobile 2025 秋季报告显示，2025 年 8 月微信小程序端整体流量约 9.50 亿。
- 艾瑞转引口径显示，2025 年中国本地服务市场规模预计约 35.3 万亿元，线上本地服务约 10.9 万亿元。

这些数字只说明赛道足够大，不代表和趣已经验证收入。

## 4. TAM / SAM / SOM

| 层级 | 讲法 | 边界 |
| --- | --- | --- |
| TAM | 私域电商和本地生活交易大盘，万亿级交易空间 | 大盘很大，但不是和趣可直接获得收入 |
| SAM | 微信小程序、社群经营、团购发布、卡券核销、本地活动/服务交易 | 更接近和趣产品可服务范围 |
| SOM | 首期可触达城市、团长、供给方和活动/团购交易规模 | 需要用真实订单、退款和复购数据逐步测算 |

现场表达：全员要记住，我们讲市场空间是为了说明方向值得验证，不是为了给还没发生的业务做收入承诺。

## 5. 用户是谁

| 用户 | 他们要完成的事 | 产品要解决的问题 |
| --- | --- | --- |
| 团长/群主 | 把社群信任变成收入 | 发品、分享、收款、售后、核销、分钱 |
| 供给方 | 把活动、商品、卡券、服务卖出去 | 上架、授权帮卖、披露责任、处理履约 |
| 推广团长 | 有社群但不一定有货 | 选品、转发、赚佣金或加价收益 |
| 消费者/团员 | 在熟人推荐下安全下单 | 订单可查、核销清楚、退款有入口 |
| 平台运营 | 控制交易风险和供给质量 | 审核、白名单、工单、结算阻断、风险暂停 |

## 6. 用户痛点

| 痛点 | 现状问题 | 和趣解法 |
| --- | --- | --- |
| 发品难 | 活动、商品、卡券、服务字段不同，长表单难用 | 商品库 + 开团类型发布 |
| 统计难 | 群内报名、付款、退款靠人工对账 | 统一订单和来源归因 |
| 履约难 | 活动核销、发货、自提、券码、预约分散 | 按履约类型管理订单 |
| 售后难 | 出问题后找不到责任方 | 售后服务台和责任协同 |
| 分钱难 | 佣金、加价、优惠、退款算不清 | 收益账本和退款冲回 |
| 复购难 | 下单后数据沉没在聊天记录 | 店铺、订单和复购经营数据沉淀 |

## 7. 产品总主线

`社群建店 -> 供给入库/选品 -> 开团发布 -> 社群分享/分销帮卖 -> 下单支付 -> 履约核销/发货/自提/券码 -> 售后退款 -> 收益结算 -> 复购经营`

这条主线是所有功能判断的尺子。任何功能如果不能帮助发布、成交、履约、售后、结算或复购，就不应该进入首发核心。

## 8. 产品形态

- 所有团长运行在同一个和趣小程序中。
- 每个团长拥有自己的专属社群店铺主页。
- 社群类型决定店铺模板和推荐货架。
- 消费者从团长主页、单品/团购页、优惠券或分享链接进入。
- 平台统一处理订单、售后、核销、收益和风控。

## 9. 核心模块总览

| 模块 | 作用 |
| --- | --- |
| 社群与店铺 | 让团长有自己的经营阵地 |
| 商品库 / 供给库 | 统一管理活动、商品、卡券和服务 |
| 开团发布 | 用不同开团类型匹配不同履约方式 |
| 分享与分销 | 让团长通过社群触达成交 |
| 订单中心 | 管理活动、物流、自提、卡券、服务预约订单 |
| 售后服务台 | 处理咨询、退款、投诉、责任协同 |
| 收益账本 | 展示待入账、可结算、冻结、退款冲回 |
| 经营数据 | 追踪开团、成交、复购、退款和团长活跃 |

## 10. 商品库 / 供给库

商品库不是只放商品，而是放所有可交易供给。

| 类型 | 示例 | 发布前必须明确 |
| --- | --- | --- |
| 活动票务 | 亲子活动、青年活动、创业活动 | 场次、票种、名额、退改、核销 |
| 实物商品 | 标准商品、低风险团购品 | 规格、库存、发货、退换 |
| 自提商品 | 社区自提、门店自提 | 自提点、提货时间、提货码 |
| 虚拟卡券 | 电子券、次卡、权益包 | 有效期、使用次数、核销规则 |
| 到店服务 | 摄影、洗护、运动体验 | 门店、预约时段、取消投诉 |

## 11. 开团发布模式

正式采用“商品库/供给库 + 开团类型”的发布方式。

| 开团类型 | 适用场景 | 首发重点 |
| --- | --- | --- |
| 普通团购 | 现货商品、标准服务、简单活动 | 快速发布、库存/名额、售后规则 |
| 预售团购 | 先收单再备货或排期 | 预售截止、预计履约、退款说明 |
| 卡券团购 | 电子券、次卡、权益包 | 券码、有效期、核销和过期规则 |
| 活动报名 | 亲子、青年、创业等活动 | 场次、票种、报名信息、保险 |
| 到店服务 | 预约核销型服务 | 门店、预约时段、服务规格 |

发布台原则：一页完成关键配置，按类型展开差异字段，预览后发布或提交审核。

## 12. 活动变现

活动是和趣最有业务基础的首期验证场景。

| 功能 | 说明 |
| --- | --- |
| 活动分类 | 亲子活动、青年兴趣、创业交流、线下体验 |
| 场次票种 | 多场次、不同票种、报名截止、名额 |
| 报名信息 | 昵称、手机号、车牌、儿童信息、保险信息 |
| 购买须知 | 退改比例、注意事项、集合地点、保险说明 |
| 履约售后 | 到场核销、活动取消、退款工单、投诉处理 |

## 13. 电商变现

电商不是简单“上商品”，重点是履约方式。

| 履约方式 | 功能 | 风险控制 |
| --- | --- | --- |
| 物流 | 规格、库存、发货、物流、退货退款 | 首发限制低风险标准品 |
| 自提 | 自提点、提货时间、提货码、核销 | 未核销退款规则前置 |
| 预售 | 预售截止、预计发货/自提、取消规则 | 不承诺复杂成团和资金监管 |

## 14. 卡券变现

卡券是连接电商和本地服务的轻量形态。

| 能力 | 说明 |
| --- | --- |
| 券包类型 | 单次券、次卡、权益包、体验券 |
| 发放方式 | 支付后生成券码或权益记录 |
| 核销方式 | 到店核销、扫码核销、人工核销 |
| 售后规则 | 未使用可退、部分使用按规则退、过期规则披露 |

## 15. 本地服务

本地服务首发只做受控补充。

| 能力 | 说明 |
| --- | --- |
| 服务上架 | 门店、服务规格、可预约时段、服务方 |
| 预约下单 | 消费者选择时段并支付 |
| 到店核销 | 服务完成后核销 |
| 取消投诉 | 支持取消、改期、未服务退款、投诉工单 |
| 禁止范围 | 上门服务、医疗美容、强效果承诺和高安全责任服务 |

## 16. 订单中心

订单按履约类型组织：

- 活动报名订单。
- 物流订单。
- 自提订单。
- 卡券核销订单。
- 服务预约订单。

订单详情统一展示：支付状态、履约状态、来源团长、优惠、实付金额、退款入口、售后进度和收益状态。

## 17. 分销帮卖

| 模式 | 规则 |
| --- | --- |
| 固定佣金分享赚 | 发布方设置固定佣金，消费者价格一致 |
| 加价帮卖 | 发布方设置底价，推广团长设置终端售价 |

边界：

- 一笔订单只有一个直接推广团长。
- 同一供给只能选择一种推广模式。
- 不做二级返佣、发展下级和层层转单。
- 推广团长不承担发布方履约责任。

## 18. 优惠券与营销

首发做：

- 店铺券。
- 单品券。
- 优惠承担方。
- 优惠与退款回退。

二期再做：

- 营销素材库。
- 朋友圈、社群、小红书、抖音口播 AI 文案草稿。
- 群接龙。
- 复购触达策略。

AI 只生成草稿，不自动外部发布。

## 19. 收益结算

| 收益项 | 说明 |
| --- | --- |
| 平台服务费 | 按消费者实付金额计提 |
| 发布方收益 | 自售金额或供给结算底价 |
| 推广收益 | 固定佣金或加价收益 |
| 优惠承担 | 记录由发布方、推广方或平台承担 |
| 退款冲回 | 全退全链取消，部分退款按比例冲回 |

收益账本状态：待入账、可结算、已结算、售后冻结、退款冲回。

## 20. 售后服务台

服务台是平台护城河。

| 能力 | 作用 |
| --- | --- |
| 咨询 | 购买前后都能联系 |
| 工单 | 退款、投诉、履约异常可记录 |
| 责任协同 | 平台客服、发布方、必要时推广方协同 |
| 结算阻断 | 售后未完成时冻结或延后收益 |
| 风险暂停 | 高退款或高投诉供给暂停开团 |

## 21. 竞品参考

快团团可参考：

- 商品库和一键开团。
- 普通团购、预售、卡券等发布思路。
- 帮卖分销。
- 分享转存。
- 团购管理和订单处理。

不能照搬：

- 全品类自由开放。
- 高复杂多级扩散。
- 未经验证的即时结算承诺。
- 只追求开团速度而忽略履约和售后。

## 22. MVP 范围

第一版必须有：

1. 社群建店。
2. 商品库/供给库。
3. 开团发布台。
4. 活动报名、普通团购、卡券团购。
5. 订单中心。
6. 售后服务台。
7. 一级分销帮卖。
8. 收益账本。

## 23. 不做清单

- 多级分销、发展下级、层层转单。
- 保证金账户、经销商层级、会员分层可见。
- 食品生鲜、医疗美容、上门服务、高风险户外、强效果承诺活动。
- 自动选货、自动上架、自动发布营销内容。
- 支付链路未核验前承诺真实自动打款。

## 24. 迭代路线

| 版本 | 目标 | 功能 |
| --- | --- | --- |
| V0 原型共识 | 全员理解产品是什么 | PPT、原型、蓝图、演示脚本 |

| V4 商单扩展 | 只对验证方向加深 | 会员经营、商单系统加强、更多品类 |

## 25. 验证指标

| 指标类型 | 核心指标 |
| --- | --- |
| 供给 | 入库供给数、审核通过率、可帮卖供给数 |
| 发布 | 开团数、发布完成率、开团类型分布 |
| 交易 | 订单数、支付转化率、客单价、来源可追踪率 |
| 履约 | 核销率、发货及时率、自提核销率、卡券使用率 |
| 售后 | 退款率、投诉率、工单时长、冻结金额 |
| 经营 | 团长留存、复购率、推广订单占比、平台服务费 |

## 26. 全员共识

和趣不是一个单点活动工具，也不是另一个普通商城。

它要成为社群交易操作系统：让有社群信任的人更容易发起交易，让消费者更放心履约和售后，让平台能够看清订单、责任、退款和收益。

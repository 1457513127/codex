## 1. 本轮目的

社区团购团长是新增验证对象，不是已确认的产品目标市场。本轮通过访谈与人工试单回答三个问题：

1. 团长是否愿意向现有社群分享低风险活动/服务，而不仅是实物团购。
2. 消费者是否会通过团长入口产生真实付费订单。
3. 平台能否在明确责任、处理退款和核算一级佣金后仍获得可验证收入。

## 3. 访谈提纲

### 当前流程

1. 最近一次开团或推荐是什么？从获得供给到成交、售后分别怎么做？
2. 你通常靠商品差价、佣金、会员费还是其他方式获得收入？
3. 现在最耗时或最容易失去群信任的是哪一步？

### 非实物活动/服务假设

4. 哪些活动或服务你愿意分享给群成员？哪些一定不会分享？为什么？
5. 对一次性体验活动，你需要先看到哪些信息才愿意发布：价格、主办方、退款、平台售后、佣金或其他？
6. 如果实际服务由主办方完成、平台处理订单售后、你只作推荐，这个责任划分是否可接受？

### 收入与试单

7. 对按真实成交获得一级佣金，你可接受的结算前提是什么？
8. 退款或投诉发生时，你希望承担什么角色，哪些责任必须由平台承担？
9. 在不改现有群运营习惯的情况下，是否愿意发布一次真实可购买的活动/服务链接？

## 5. 人工试单方案

### 5.1 试单边界

| 项目 | 规则 |
| --- | --- |
| 团长数量 | 3-5 名阶段 0 通过者 |
| 供给数量 | 1-2 个低风险活动/服务 SKU |
| 供给优先级 | 优先采用现有可控亲子活动中适配团长成员的项目；必要时人工审核一个低风险本地体验 |
| 分发关系 | 仅平台/供给方 -> 团长 -> 消费者 |
| 售后处理 | 平台统一入口，人工登记和处理 |
| 佣金处理 | 使用明确规则人工核算，不做自动分账或保证金 |
| 禁止事项 | 实物团购、多级转单、团长任意加价、下级招募 |

## 6. 试单通过与停止判断

### 通过条件

- 出现真实支付订单，且可追踪到具体团长和分享入口。
- 履约、退款和投诉能通过人工流程完整闭环。
- 能核算平台收入、一级佣金与售后人工投入。
- 至少一名团长在完成一次发布后愿意继续分享同类项目。

### 立即停止或复盘

- 团长提出必须转卖给下级、隐藏责任关系或任意加价才愿意参与。
- 供给项目无法给出清晰退款/履约责任。
- 退款、投诉或人工处理成本超过可承受范围。
- 试单仅证明团长想卖商品，未证明活动/服务交易价值。

| 输出 | 内容 |
| --- | --- |
| 访谈结论 | 访谈数、愿试单数、抽佣接受数、主要拒绝原因 |
| 交易结论 | 分享数、订单数、支付金额、履约/退款/投诉结果 |
| 经济性结论 | 平台收入、一级佣金、售后耗时和初步贡献判断 |

| 明确不做 | 本轮被证据否定或仍不可进入的功能/品类 |

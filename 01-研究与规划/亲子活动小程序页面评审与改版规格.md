# 亲子活动小程序页面评审与增长阶段改版规格

版本：V2.0 增长阶段修订版  
日期：2026-05-26  
适用范围：现有 `创建团购商品`、`创建活动`、`工作台` 页面，以及新增团长主页和推广订单能力  
关联规划：[亲子活动交易与受控分销平台产品规划](./私域线下体验活动交易留痕平台产品规划.md)

## 1. 改版判断

亲子活动交易已进入增长验证阶段，界面必须从“单一活动组织者工作台”扩展为区分角色的产品路径：

| 身份 | 首期主要页面 |
| --- | --- |
| 主办方/公司运营 | 活动创建、审核、履约和售后处理 |
| 帮卖团长 | 我的主页、可推广活动、推广订单、佣金明细 |
| 家长 | 团长主页、活动详情、报名支付、订单售后 |

本次调整恢复的不是泛营销平台，而是**受控一级分销**。已有商品、优惠券、多级邀请、内容社区仍然不恢复。

## 2. 全局导航与功能开关

### 2.1 角色入口

| 用户 | 首页/工作台首要入口 | 次要入口 |
| --- | --- | --- |
| 主办方/公司运营 | 活动管理、待审核/待退款/待核销、发布活动 | 数据、主办方质量、允许分销管理 |
| 团长 | 我的主页、可推广活动、推广订单、佣金明细 | 消息、规则说明 |
| 家长 | 团长主页或活动详情 | 我的订单、退款/申诉 |

若同一账号兼具主办方和团长资格，应提供身份切换，不在同一首屏混排履约待办与推广收入。

### 2.2 功能处置总表

| 当前/候选能力 | 当前阶段处理 | 备注 |
| --- | --- | --- |
| 创建活动 | 保留并按亲子字段管理 | 面向公司/主办方运营，不由帮卖团长创建外部供给 |
| 创建团购商品 | 隐藏 | 商品另行立项 |
| 商品 Tab | 隐藏 | 首轮仅亲子活动 |
| 团长主页 | 新增 | 轻主页，不做内容社区 |
| 可推广活动 | 新增 | 只能选择审核允许活动 |
| 推广订单与佣金 | 新增 | 仅一级佣金，显示状态和退款影响 |
| 一键开团 | 不恢复 | 改为 `选择推广活动` 或 `发布活动`，随身份展示 |
| 邀请赚 | 不恢复 | 不发展下级团长 |
| 优惠券、门窗管理、社群资料 | 隐藏 | 与本轮验证无关 |

## 3. 页面一：创建团购商品

### 3.1 判断

继续隐藏。商品名称、规格、库存、限购和收货/售后字段会开启供货、物流与商品质量责任，无法由亲子活动分销试点验证。

### 3.2 规则

| 项目 | 处理 |
| --- | --- |
| 工作台入口 | 不显示 |
| 路由直达 | 显示功能未开放并返回当前身份首页 |
| 代码组件 | 可留存，但不继续投入接口与交互开发 |
| 重新启用 | 需单独完成商品方向立项、供应链和售后评审 |

## 4. 页面二：活动创建与供给审核

### 4.1 页面身份

原页面改为 `创建亲子活动`，供公司自营活动运营或经允许的主办方使用。**帮卖团长不因取得推广资格而获得活动创建/修改主办方资料权限。**

### 4.2 四步结构

| 步骤 | 页面内容 | 增长阶段新增 |
| --- | --- | --- |
| 1. 基本信息 | 名称、封面、详情、适龄、地点、陪同要求 | 供给类型、自营/外部精选、实际主办方 |
| 2. 场次票种 | 多场次、儿童票、成人陪同票、价格、名额 | 售价、主办方结算/成本字段仅运营可见 |
| 3. 保障规则 | 安全、退款、监护授权说明、参与人保险、异常联系 | 退款窗口、是否允许进入分销审核 |
| 4. 审核发布 | 家长可见预览、通知群、售后说明、提交审核 | 分销状态与可推广佣金规则单独审核 |

### 4.3 字段与文案变更

| 当前字段/缺口 | 改为/新增 | 展示范围 |
| --- | --- | --- |
| `创建组团活动` | `创建亲子活动` | 运营/主办方 |
| `分组设置` | `票种设置` | 运营及家长摘要 |
| 团长为组织者 | `实际主办方` | 家长付款前可见 |
| 无供给来源 | `供给类型：公司自营 / 外部精选代售` | 运营可见，必要说明对家长可见 |
| `保险选择` | `参与人保险` | 家长可见摘要并逐人选择 |
| `活动群二维码` | `报名后通知群（可选）` | 报名成功后展示 |
| 无分销审批 | `允许团长推广`、佣金规则、退款窗口 | 运营审核，不在普通创建者随意配置 |

### 4.4 分销审核状态

| 状态 | 说明 | 可操作 |
| --- | --- | --- |
| 不允许推广 | 仅平台/自有入口销售 | 运营评估后可申请开放 |
| 待分销审核 | 活动交易已通过，等待供给质量与佣金规则检查 | 不进入团长货架 |
| 可推广 | 团长可选择至主页 | 运营可暂停 |
| 已暂停推广 | 高退款、取消、投诉或资料变更 | 已有订单照原规则处理 |

## 5. 页面三：工作台按身份拆分

### 5.1 原页面需要改掉的信号

| 当前显示 | 改版处理 |
| --- | --- |
| `商品 / 活动 / 草稿箱` | 主办方工作台为 `活动 / 草稿`；团长首页不出现商品 |
| `一键开团` | 主办方显示 `发布活动`；团长显示 `选择推广活动` |
| `团长结算佣金` 混在自办活动卡 | 自办活动显示 `待结算金额`；推广身份单独显示 `推广佣金` |
| `邀请赚`、`优惠券`、`门窗管理`、`社群资料` | 不显示 |

### 5.2 主办方/运营工作台

首屏仍以履约待办为中心：

| 区域 | 内容 |
| --- | --- |
| 身份栏 | `主办方/运营` 标识、名称、审核状态、切换身份 |
| 今日待办 | 待审核活动、待处理退款、今日待核销、保险异常、待确认完成 |
| 快捷操作 | 发布活动、扫码核销、订单处理、退款/申诉 |
| 活动筛选 | 草稿、审核中、报名中、待开始、进行中、已完成、暂停分销/已下架 |
| 活动卡 | 主办方、场次、报名、退款、保险、核销、是否可推广、待结算金额 |

### 5.3 团长推广首页

团长不处理现场履约任务，首屏围绕推荐与订单反馈：

| 区域 | 内容 | 主操作 |
| --- | --- | --- |
| 身份栏 | 团长名称、审核状态、主页状态 | `查看我的主页` |
| 佣金概览 | 预估佣金、待生效、可结算、已结算/回退入口 | `佣金明细` |
| 推广任务 | 在售推广活动数、即将结束报名、退款影响订单 | `处理/查看` |
| 快捷操作 | 编辑主页、选择活动、分享主页、推广订单 | 对应页面 |
| 活动货架管理 | 已上架、可选择、暂停推广 | 上架/下架、分享 |

不显示给团长的能力：

- 活动现场核销和主办方退款裁定入口，除非该账号另有主办方身份并切换进入。
- 招募其他团长、二级佣金、商品选品和营销优惠券。

## 6. 新增页面：团长轻主页（家长可见）

### 6.1 页面目标

让家长在微信分享落地后，能在一个页面确认“谁推荐、谁履约、有哪些活动能报名、售后在哪里”，并让订单产生可核验团长归因。

### 6.2 信息架构

| 顺序 | 模块 | 内容/文案要求 |
| ---: | --- | --- |
| 1 | 团长身份 | 头像、团长/社群名、审核标识、推荐说明 |
| 2 | 关系披露 | `本页活动由团长推荐；具体活动由页面标明的主办方提供；支付与售后请通过和趣订单完成` |
| 3 | 可报名活动 | 仅可推广的在售亲子活动，可按最近场次/适龄筛选 |
| 4 | 活动卡片 | 封面、名称、适龄、最近场次、起价、主办方、保险提示、退款摘要 |
| 5 | 信任说明 | 退款/申诉入口、儿童信息保护和保险说明链接 |

### 6.3 交互规则

- 分享主页和活动卡均生成 `leader_id` 与入口来源；家长无需手动选择推荐人。
- 页面只显示审核通过且仍可报名/推广的活动，暂停活动立即撤下。
- 不出现动态、评论、成员数量营销、商品或优惠券组件。

## 7. 新增页面：可推广活动与推广订单

### 7.1 可推广活动

| 模块 | 内容 |
| --- | --- |
| 筛选 | 最近可报名、适龄、品类、已上架/未上架 |
| 卡片 | 标题、场次、价格、主办方、退款摘要、保险、预估佣金规则、风险状态 |
| 操作 | 上架主页、分享活动、查看详情 |
| 风险限制 | 暂停推广或高风险活动不可上架/分享 |

### 7.2 推广订单

| 字段 | 说明 |
| --- | --- |
| 订单摘要 | 脱敏订单号、活动、场次、下单时间 |
| 状态 | 已支付、已核销、已完成、退款中、已退款 |
| 佣金 | 预估/待生效/可结算/已结算/已回退与金额 |
| 退款影响 | 明示退款导致佣金取消、回退或待处理 |
| 隐私 | 团长不得查看儿童和保险敏感明细，仅看与佣金及状态相关必要信息 |

## 8. 家长报名与订单改动

| 页面 | 增长阶段必须增加 |
| --- | --- |
| 团长来源活动详情 | 展示推荐团长、实际主办方、平台售后关系 |
| 报名支付 | 继续保留适龄、参与人、监护授权、逐人保险、退款规则确认 |
| 订单详情 | 显示来源团长（若有）、主办方、规则快照、售后和通知群 |
| 退款申请 | 显示退款将按规则处理；佣金处理不转嫁给家长 |

## 9. 运营后台补充

| 页面/能力 | 要求 |
| --- | --- |
| 主办方质量 | 取消、退款、核销、投诉、保险异常，允许/暂停分销 |
| 团长审核 | 资格、主页、违规状态、分享与成交质量 |
| 佣金账本 | 活动规则、订单归因、佣金状态、结算批次、回退/冲抵 |
| 渠道看板 | 自有与团长渠道的访问、支付、核销、退款、毛利对比 |
| 数据导出 | 支持对账和试点复盘，关键字段完整 |

## 10. 实施顺序

| 优先级 | 改造项 | 验收重点 |
| --- | --- | --- |
| P0 | 角色模型和字段升级：主办方、团长、供给类型、归因、佣金状态 | 旧订单兼容，责任展示正确 |
| P0 | 数据口径和支付/结算方案确认 | 不带着不可回退佣金扩量 |
| P0 | 团长轻主页与专属分享归因 | 主页至支付路径可追踪 |
| P0 | 推广订单与佣金账本 | 退款状态能影响佣金 |
| P1 | 活动创建增加外部供给与分销审核 | 仅合格活动进入货架 |
| P1 | 双身份工作台 | 履约和推广任务不混淆 |
| P2 | 团长经营数据与主办方质量分层 | 为 Next 扩量决策准备 |

## 11. 验收清单

- 家长从团长主页看到团长为推荐者、实际主办方和平台售后入口。
- 团长仅能推广审核通过的亲子活动，不能创建商品或未开放服务。
- 家长下单后订单记录唯一团长归因，同时保留原有儿童授权、保险和规则快照。
- 团长能查看推广订单和佣金状态，但不能查看儿童敏感信息。
- 履约结束且超过退款窗口后佣金才可结算；退款形成取消或回退/冲抵处理记录。
- 主办方/运营工作台仍优先展示退款、核销和保险异常等履约待办。
- 工作台不出现商品、优惠券、多级邀请或内容社区入口。

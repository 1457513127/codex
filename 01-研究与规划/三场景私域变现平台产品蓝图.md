# 三场景私域变现平台产品蓝图

版本：V1.0

日期：2026-05-27

状态：当前权威产品规划入口

## 1. 产品定位

和趣将建设面向**经营型团长**的私域变现平台。团长可以运营自己的店铺、发布供给，也可以选择其他经营者开放帮卖的供给进行一级推广；消费者在平台完成购买、交付确认与售后。

首轮产品蓝图覆盖三类交易：

| 场景 | 首轮能力 | 暂缓能力 |
| --- | --- | --- |
| 活动 | 发布活动、报名、核销、取消/退款、一级帮卖 | 复杂多日行程、高风险服务扩量 |
| 电商商品 | 发布商品、库存/可售、发货/物流、退换售后、一级帮卖 | 平台自营仓配、开放供应链撮合 |
| 付费内容/课程包 | 发布内容商品、支付后授权入口、交付状态与退款 | 直播、训练营、打卡、互动教学 |

本地服务和广告变现不进入本轮蓝图。多级分销、发展下级返佣和层层转单明确禁止。

## 2. 项目会表达口径

本次项目会用于产品负责人同步产品方向、功能版图和分期路线，并给开发、运营、支付/合规明确前置任务；不以现场 `Go / No-Go` 讨论作为主旨。

当前承诺为：

- 三场景方向已定，开发可并行拆解各场景模块和共用底座。
- 真正上线顺序由支付、履约、售后、商品责任、内容授权等可行性结果控制。
- 当前仓库交付的是功能规划、接口预研规格与可点击原型，不代表真实后端或自动打款已经可用。

## 3. 核心角色

| 角色 | 能力 | 责任边界 |
| --- | --- | --- |
| 消费者 | 浏览店铺、支付、接收活动/商品/内容交付、申请售后 | 按披露规则使用服务并提交真实售后信息 |
| 经营型团长 | 开店、订阅经营工具、发布自己的供给、选择他人供给帮卖、查看订单与收益 | 作为发布者时承担对应供给履约责任；作为推广者时披露推荐关系 |
| 平台运营 | 审核账号与供给、配置服务费和套餐、处理争议、监控结算与风险 | 建立审核和处置规则，不无条件代替供给方履约 |
| 支付/合规人员 | 复核结算、退款回退、主体资质和协议披露 | 自动结算上线前确认既有支付链路可复用 |

团长在同一账号下可以同时拥有“供给发布者”和“一级推广者”身份，但一张订单最多记录一个发布者和一个直接推广者。

## 4. 通用产品底座

| 模块 | 当前规划能力 | 开发预研重点 |
| --- | --- | --- |
| 账号与店铺 | 团长认证、套餐状态、店铺信息、三类货架 | 角色组合、权限隔离、订阅状态 |
| 供给中心 | 创建、审核、上下架、允许帮卖、责任披露 | 通用字段与场景扩展结构 |
| 帮卖与归因 | 选择供给、设置终端售价、专属分享、唯一一级归因 | 定价约束、归因锁定与冲突处理 |
| 订单与交付 | 下单、支付、三场景状态、退款/售后 | 场景状态机、订单规则快照 |
| 收益与结算 | 底价、平台费、推广收益、订阅、待结算/回退 | 既有结算链路复用、部分退款回退 |
| 运营治理 | 供给审核、投诉工单、结算异常、数据看板 | 敏感信息权限、风险暂停能力 |

## 5. 场景功能规格

### 5.1 活动

| 能力 | 规则 |
| --- | --- |
| 发布 | 标题、时间、地点、名额、履约方、价格、退改规则、核销方式 |
| 交付 | 消费者报名支付后获得报名凭证，活动完成或取消驱动订单状态 |
| 售后 | 取消、退款和投诉关联活动场次及责任方 |
| 帮卖 | 可由发布团长开放一级推广，推广团长可在最低可售价之上加价 |

### 5.2 电商商品

| 能力 | 规则 |
| --- | --- |
| 发布 | 商品信息、规格、底价、库存/可售状态、发货时效、退换规则 |
| 交付 | 发布商品的团长/商家负责发货、物流和商品售后 |
| 平台职责 | 记录交易、展示责任归属并提供争议处理入口 |
| 帮卖 | 推广团长只负责推荐与定价，不承担商品发货或售后责任 |

### 5.3 付费内容/课程包

| 能力 | 规则 |
| --- | --- |
| 发布 | 标题、内容说明、交付入口、底价、授权期限/范围、退款规则 |
| 交付 | 消费者支付后获得受控内容或课程包访问入口，记录授权状态 |
| 售后 | 处理未交付、入口失效、内容不符及规则允许的退款 |
| 暂缓 | 直播、训练营、社群打卡和服务型辅导不进入首轮 |

## 6. 定价、收入与结算规则

### 6.1 收入模型

| 收入 | 付费方 | 说明 |
| --- | --- | --- |
| 工具订阅费 | 经营型团长 | 用于店铺、经营工具、数据与分发能力 |
| 交易服务费 | 订单交易相关方 | 平台按消费者实际支付的终端成交额计提 |

### 6.2 一级帮卖定价

- 发布团长设置供给结算底价和是否允许帮卖。
- 推广团长可设置终端售价并获得加价空间，但仅可直接推广一次，不可再转给下级。
- 终端售价不得低于 `供给结算底价 + 平台按终端成交额计算的最低服务费`。
- 消费者支付前必须看到实际供给/履约责任方和推荐关系。

### 6.3 收益计算

| 场景 | 发布方应结算 | 推广方收益 | 平台交易服务费 |
| --- | ---: | ---: | ---: |
| 发布团长自售 | 终端成交额 - 平台服务费 | 0 | 按终端成交额计提 |
| 一级帮卖成交 | 供给结算底价 | 终端成交额 - 供给结算底价 - 平台服务费 | 按终端成交额计提 |

### 6.4 退款与自动结算

- 全额退款时，发布方应结算、推广方收益与平台交易服务费全部回退或取消。
- 部分退款时，三方对应收益按退款金额占终端成交额的比例冲回。
- 自动结算是目标能力；上线前必须确认既有运行结算链路支持三类供给、双角色账户、部分退款回退与订阅/抽佣并存。
- 在复用核验通过前，原型和规格可以展示自动结算状态，真实资金打款不得作为当前已交付能力宣称。

## 7. 数据对象与接口边界

| 对象 | 最小字段/含义 |
| --- | --- |
| `LeaderAccount` | 团长主体、认证状态、发布/推广权限、结算账户状态 |
| `Storefront` | 店铺、订阅套餐、三场景货架和公开信息 |
| `Offering` | `activity/goods/knowledge` 类型、发布者、底价、允许帮卖、审核/上下架、售后责任 |
| `DistributionListing` | 供给、推广团长、终端售价、分享状态、有效期 |
| `Attribution` | 消费者入口、直接推广团长、锁定时间和归因规则版本 |
| `Order` | 购买者、供给、发布者、推广者、终端成交额、场景交付状态、退款状态 |
| `SettlementLedger` | 底价结算、推广收益、平台费、退款冲回、结算批次和状态 |
| `SubscriptionPlan` | 套餐、周期、可用经营能力、订阅订单和状态 |
| `AfterSalesCase` | 订单、问题类型、责任方、处理节点、退款/争议结论 |

开发预研不需要立即落库，但必须给出对象复用方式、各场景新增字段、现有支付/订单能力可复用点和不能复用点。

## 8. 分期路线与团队任务

| 阶段 | 产品交付 | 开发任务 | 运营任务 | 上线门槛 |
| --- | --- | --- | --- | --- |
| A：蓝图与底座规格 | 三场景产品蓝图、界面原型、对象与规则 | 数据对象、接口、权限、结算复用评估 | 目标团长和三类供给清单 | 规划一致、风险清单明确 |
| B：场景模块预开发 | 页面流与状态细化 | 活动/商品/内容模块并行拆解和技术方案 | 各场景供给责任、定价和售后事实核实 | 可形成排期和依赖 |
| C：可行性门槛后开放 | 按验证结果确定首批真实入口 | 对通过门槛的模块开发/联调 | 执行招募与运营验证 | 支付、履约、售后与合规通过 |
| D：后续扩展 | 本地服务、广告或复杂内容另案规划 | 按新规则评估 | 新市场研究 | 既有场景经营数据成立 |

## 9. 可行性核验清单

| 责任方 | 会后必须给出的结论 |
| --- | --- |
| 开发 | 三场景对象模型、一级加价帮卖、状态机、权限与数据导出的预研结论 |
| 支付/财务 | 既有自动结算链路能否支持双角色、终端额抽佣和按比例退款回退 |
| 运营 | 活动、商品、知识内容各自的首批供给来源、履约责任和目标团长样本 |
| 合规/负责人 | 商品责任、内容授权、推荐披露、订阅协议与争议处理边界 |

## 10. 不纳入本轮的能力

- 多级分销、招募返佣、价格链路多次转单。
- 本地维修/到家等服务型供给。
- 广告投放或流量分成。
- 直播、训练营、打卡与群内重互动。
- 支付链路复用核验未完成前的真实自动打款上线。

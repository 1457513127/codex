# 和趣私域社群变现平台产品蓝图

版本：V3.4

日期：2026-05-28

状态：当前权威产品规划入口

## 1. 一句话定位

和趣是面向私域社群经营者的社群变现平台，服务已有社群经营者和零基础新手群主两类核心客户，帮助他们把微信群、兴趣圈层、本地社区里的信任关系转化为可发布、可成交、可履约、可复购的社群生意。

平台核心价值不是单纯“建一个店”“找一批货”或“让团长帮卖”，而是给不同成熟度的社群经营者一套从经营方向、社群承接、可信交易到复购经营的路径，并把社群交易中最容易失控的供给准入、成交归因、履约责任、售后协同和收益解释纳入可监督的产品闭环。

当前版本定位仍处于产品策划调整期，以下口径作为稳定方向沉淀，不把未确认的具体功能细节提前写死：

- 和趣不是单一商品卖货工具，而是“社群经营者 + 可信供给 + 消费者订单 + 平台运营监督”的交易组织平台。
- 和趣优先解决“信任在群里、交易和履约却不可管理”的问题，而不是优先追求全品类货盘或多级扩散。
- 已有社群经营者关注经营效率、供给组织和履约可信；零基础新手群主关注做什么、服务谁、如何启动第一笔真实交易。
- 供给方负责供给真实性、交付能力和基础履约承诺；消费者通过社群关系或内容入口下单，但需要平台化的规则、凭证和售后入口；平台运营负责准入审核、风险控制、纠纷协同和账本监督。

## 2. 赛道与市场口径

和趣进入的是三层叠加赛道：

| 层级 | 赛道 | 和趣切入方式 |
| --- | --- | --- |
| 主赛道 | 私域社群变现 / 社群团购 | 团长、群主、主理人通过微信群、小程序和社群信任关系完成交易转化 |
| 延展赛道 | 本地生活交易 | 通过活动报名、到店服务、卡券核销承接线下履约 |
| 首期切入口 | 社群冷启动、活动报名、普通团购、卡券团购 | 先帮助新手完成第一团，并用低风险白名单供给验证交易闭环 |

### TAM / SAM / SOM

| 层级 | 估算口径 | 当前使用方式 |
| --- | --- | --- |
| TAM 全国大盘 | 私域电商 + 本地服务交易大盘。艾瑞咨询《2025年中国私域电商行业趋势白皮书》显示，私域电商 2024 年约 2.3 万亿元，2025 年预计突破 3 万亿元；本地服务市场 2025 年预计约 35.3 万亿元，其中线上本地服务约 10.9 万亿元。 | 说明赛道足够大，但不直接等同于和趣可获得收入 |
| SAM 可服务市场 | 微信小程序、社群经营、团购发布、卡券核销、本地活动/服务交易。QuestMobile 2025 秋季报告显示，2025 年 8 月微信小程序端整体流量约 9.50 亿。 | 说明小程序和社群交易是可触达基础 |
| SOM 首期可切入市场 | 首期可触达城市、已有团长、新手群主、供给方和活动/团购交易规模。以和趣现有活动供给、团长资源、新手冷启动模板、亲子家庭/青年兴趣/创业职场/社区生活等活动和低风险商品卡券为起点。 | 只用假设区间做经营测算，需靠真实订单验证 |

市场数据只用于证明“值得进入并验证”，不证明和趣已经拥有市场份额、付费率或 ROI。

## 3. 产品总主线

`选择经营方向 -> 社群冷启动 -> 建店 -> 商品库/供给库 -> 开团发布 -> 分享成交 -> 履约售后 -> 收益结算 -> 复购经营 -> 选品中心放大`

| 环节 | 首发能力 | 关键产出 |
| --- | --- | --- |
| 选择经营方向 | 团长注册、选择社群类型和兴趣标签，明确亲子家庭、青年兴趣、创业职场、社区生活等方向，并预留文旅休闲、健康运动、学习成长 | 新手知道自己要服务谁 |
| 社群冷启动 | 新手店铺模板、首单开团清单、邀请成员、首单转化和复盘路径 | 零基础群主能从第一团开始 |
| 社群建店 | 生成专属社群主页、货架和分享入口 | 团长有自己的经营阵地 |
| 供给入库/选品 | V1 使用平台审核货盘和人工推荐；V2 建设选品中心 | 供给来源和责任方清楚 |
| 开团发布 | 普通团购、预售团购、卡券团购、活动报名、到店服务 | 发布方式匹配履约方式 |
| 分享成交 | 店铺、单品、团购页、优惠券分享，记录唯一直接推广方 | 订单来源可追踪 |
| 订单履约 | 活动核销、物流发货、自提、卡券核销、到店预约核销 | 消费者知道如何获得服务 |
| 售后结算 | 售后服务台、退款工单、收益账本、结算阻断 | 钱和责任都可解释 |
| 复购经营 | 浏览、订单、卡券、活动记录和优惠触达 | 后续运营有数据基础 |
| 选品中心放大 | V2 提供审核货盘分层、推荐供给包、素材包和经营数据 | 新手和成熟团长都能放大生意 |

## 4. 产品承载方式：小程序双通道与多端入口

本节只说明产品如何承载业务，不作为项目主题。项目标题和开场应始终围绕“私域社群变现”，多端不是多套系统，和趣的承载方式是“统一交易底座 + 多端入口”。

| 层级 | V1 形态 | 后续形态 |
| --- | --- | --- |
| 统一交易底座 | 账号、社群店铺、商品库/供给库、开团发布、订单、售后、收益、归因共用 | 持续作为所有端的唯一业务底座 |
| B 端经营通道 | 同一微信小程序内提供团长/群主经营台 | 团长 App 承接选品中心、开团管理、订单售后、收益账本、经营数据和复购运营 |
| C 端消费通道 | 微信小程序消费端承接社群店铺、团购页、活动页、卡券页、订单、核销和售后 | 扩展微信、小红书、抖音小程序消费端，共用货盘、订单、售后和结算 |

V1 不拆两个小程序；同一用户可按身份进入 C 端消费或 B 端经营。后续团长 App 是专业经营工具，不替代 V1 小程序承载方式；小红书和抖音小程序是消费承接入口，不单独拆货盘、订单或结算。

## 5. 目标用户与角色

和趣的用户分层先按“经营成熟度”和“交易角色”理解，不把所有人都混称为团长：

- 已有社群经营者：已经有微信群、会员、社区、兴趣圈或本地客户触达，问题通常不是从零建群，而是供给发布不规范、成交统计分散、履约售后难解释、复购经营弱。
- 零基础新手群主：可能只有兴趣、人脉或本地资源，还没有稳定社群和供给，问题是缺少经营方向、社群承接方式、第一团路径和可信供给。
- 供给方：提供活动、商品、卡券或到店服务，可以自己经营社群，也可以授权他人帮卖；其核心边界是对供给真实性、履约能力和基础售后规则负责。
- 消费者/团员：因为熟人信任、社群推荐或内容种草下单，核心诉求是知道买什么、谁负责、如何核销/收货/到店、出问题去哪处理。
- 平台运营：不替供给方无限兜底，也不替群主承诺销售结果；核心责任是制定准入和交易规则，维护白名单、风险暂停、售后协同和收益监督。

| 角色 | 核心需求 | 和趣提供的能力 |
| --- | --- | --- |
| 已有社群经营者 | 已有群、用户和触达，希望提升成交效率和履约可信度 | 社群店铺、商品库、开团发布、分销帮卖、订单、售后和收益管理 |
| 零基础新手群主 | 没有成熟社群或供给，希望知道做什么、怎么建群、怎么开第一团 | 社群类型选择、冷启动任务路径、新手店铺模板、首批推荐场景和首单开团清单 |
| 供给发布方 | 发布活动、商品、卡券或到店服务，并获得销售渠道 | 供给入库、审核、履约规则、帮卖授权、结算账本 |
| 推广团长 | 不一定有自有供给，但有社群触达能力 | 选品帮卖、固定佣金、加价帮卖、推广订单和收益 |
| 消费者/团员 | 在熟人、内容种草或短视频兴趣流量推荐下下单，但需要确定履约和售后 | 微信小程序消费端优先承接订单、购买须知、核销凭证、售后退款入口，后续扩展小红书/抖音小程序消费端 |
| 平台运营 | 控制供给质量、交易风险和纠纷处理 | 审核、白名单、订单服务台、风险暂停和账本监督 |

### 已有社群经营者与零基础新手群主的差异

| 维度 | 已有社群经营者 | 零基础新手群主 |
| --- | --- | --- |
| 起点 | 已有社群、客户关系或持续触达 | 只有兴趣、人脉、资源线索或本地生活半径 |
| 核心问题 | 成交和履约分散在群聊、表格、私聊和人工对账中 | 不知道服务谁、卖什么、如何组织第一批成员和第一单 |
| 产品价值 | 提升发布效率、订单可追踪、售后可协同、收益可解释 | 降低起步门槛，给出经营方向、社群承接、首单路径和可信供给 |
| 风险点 | 供给扩张过快、售后责任不清、推广归因争议 | 过早承诺自动化和大收益，导致社群未建立就进入复杂交易 |
| 当前版本假设 | 可作为首批验证的成熟经营样本 | 可作为冷启动路径和第一团转化的验证样本 |

### 角色边界

| 角色 | 应负责 | 不应默认负责 |
| --- | --- | --- |
| 供给方 | 供给真实性、价格/库存/名额、履约说明、基础售后承诺 | 平台流量保证、所有推广转化结果、消费者超出规则的个性化诉求 |
| 社群经营者/群主 | 社群信任经营、选品判断、内容分享、成员沟通和交易触达 | 替供给方承担未披露的履约责任，或替平台承担支付/结算合规责任 |
| 推广团长 | 按授权规则推广、说明价格和售后边界、维护直接推广订单关系 | 发展下级、多级转单、对非本人推广订单主张收益 |
| 消费者/团员 | 按购买须知下单、提供必要履约信息、在规则内发起售后 | 要求社群私聊替代平台订单和售后规则 |
| 平台运营 | 供给准入、规则配置、风险暂停、纠纷协同、账本监督 | 未经核验前承诺自动打款，或替供给方承诺所有履约结果 |

## 6. 新手社群冷启动

新手群主是核心客户之一，但首发的“起号”只指社群冷启动，不进入小红书、抖音或视频号公域账号代运营。

| 能力 | 首发说明 |
| --- | --- |
| 社群类型选择 | 团长选择亲子家庭、青年兴趣、创业职场、社区生活、文旅休闲、健康运动、学习成长等主方向和兴趣标签 |
| 新手店铺模板 | 按社群类型生成首页模块、推荐货架顺序和内容提示 |
| 首批推荐场景 | 优先推荐活动报名、普通团购、卡券团购，降低第一单履约复杂度 |
| 首单开团清单 | 标题、图片、价格、库存/名额、售后规则、推广方式、分享入口 |
| 新手任务路径 | 建店、选品、发布、邀请、首单、复盘，帮助用户知道下一步做什么 |

V1 不做自动选货、自动上架或自动外部发布营销内容；新手可以使用平台审核货盘和人工推荐完成第一团。

## 7. 商品库 / 供给库模式

发布功能正式从“单独创建活动 / 单独创建商品”改为“商品库 + 开团类型”。

| 供给来源 | 含义 | 首发规则 |
| --- | --- | --- |
| 自有供给入库 | 团长或供给方创建活动、商品、卡券或服务 | 提交平台审核后才能开团 |
| 平台审核货盘 | 平台运营维护可售供给 | V1 可按社群类型人工推荐给团长 |
| 帮卖供给 | 发布方允许其他团长推广的供给 | 必须明确佣金、底价、售后责任和退款规则 |

商品库首发分类：

- 活动票务：亲子家庭、青年兴趣、创业职场、社区生活等首发白名单活动，以及文旅休闲、健康运动、学习成长等受控扩展活动。
- 实物商品：低风险标准商品。
- 自提商品：社区或门店自提。
- 虚拟卡券：电子券、次卡、权益包。
- 到店服务：预约核销型本地服务。

活动货盘按风险分层管理：

| 层级 | 活动方向 | 示例 | 首发边界 |
| --- | --- | --- | --- |
| 首发白名单 | 亲子家庭 | 亲子手作、研学体验、儿童剧/展览、亲子运动、营地轻体验 | 标准场次、票种、核销、退改和安全责任可说明 |
| 首发白名单 | 青年兴趣 | Citywalk、桌游社交、摄影采风、手作体验、音乐/观影活动、轻运动体验 | 轻组织、轻履约，不承诺强社交或强结果 |
| 首发白名单 | 创业职场 | 创业沙龙、资源对接、项目路演、行业交流、技能工作坊 | 只做活动组织和报名履约，不承诺融资、收益或资源结果 |
| 首发白名单 | 社区生活 | 邻里集市、社区体验课、便民活动、节日主题活动、社区亲子日 | 适合新手社群冷启动和本地复购 |
| 受控扩展 | 文旅休闲 | 周边短途、非遗体验、博物馆/展馆讲解、城市文化路线 | 先做轻路线和单日活动，多日复杂行程暂缓 |
| 受控扩展 | 健康运动 | 瑜伽、羽毛球、骑行、徒步轻路线、运动体验课 | 先做低风险体验，不做高风险户外或医疗健康承诺 |
| 受控扩展 | 学习成长 | 读书会、公开课、技能体验、亲子教育讲座 | 只做体验和知识分享，不做强效果承诺培训 |

暂不开放的活动包括：高风险户外、多日复杂行程、强效果承诺培训、医疗健康承诺、金融投资承诺、未成年人高风险托管类活动。

不允许未审核供给直接进入社群分享成交。

## 8. 选品中心

选品中心是 V2 增强版能力，用来帮助新手群主和成熟团长放大生意，不进入 V1 必做范围。

| 能力 | V2 说明 |
| --- | --- |
| 审核货盘分层 | 活动、实物、自提、卡券、到店服务分层管理 |
| 社群推荐供给包 | 按亲子家庭、青年兴趣、创业职场、社区生活、文旅休闲、健康运动、学习成长等类型推荐可开团供给 |
| 新手直接选卖 | 新手可选择成熟供给开团，降低自建供给门槛 |
| 经营辅助 | 后续加入热卖榜、转化数据、推荐理由、素材包和经营建议 |

## 9. 开团发布模式

开团发布台是团长的核心工作台。

| 开团类型 | 适用供给 | 必填配置 | 履约方式 |
| --- | --- | --- | --- |
| 普通团购 | 标准商品、标准活动、低风险服务 | 标题、图片、价格、库存/名额、售卖时间、售后规则 | 直接下单后按规则履约 |
| 预售团购 | 需要先收单再备货或排期的商品/服务 | 预售截止、预计履约时间、退款说明、供给责任 | 先锁定需求，后发货/自提/核销 |
| 卡券团购 | 电子券、次卡、权益包、体验券 | 有效期、适用门店、使用次数、核销规则、过期规则 | 支付后发放券码，到店或线上核销 |
| 活动报名 | 亲子家庭、青年兴趣、创业职场、社区生活等白名单活动，受控扩展文旅休闲、健康运动、学习成长 | 场次、票种、报名信息、保险、退改规则、核销方式 | 报名后到场核销 |
| 到店服务 | 摄影、洗护、运动体验、亲子配套服务 | 门店、服务规格、预约时段、取消规则、投诉规则 | 预约到店后核销 |

发布台不采用冗长分步骤表单，而是“一页完成关键配置 + 按开团类型展开差异字段 + 预览后发布/提交审核”。

## 10. 变现场景路线

| 阶段 | 重点场景 | 目标 |
| --- | --- | --- |
| 首期 V1 | 私域社群变现交易闭环 + 新手冷启动基础 | 同一微信小程序内跑通消费者下单与团长经营，帮新手完成第一团 |
| 第二期 V2 | 选品中心 + 预售/服务履约增强 | 用审核货盘、推荐供给包和成熟供给选卖帮助团长放大，同时验证预售和到店服务 |
| 第三期 V3 | 团长 App + 经营效率工具 | App 承接重度经营、选品中心、订单售后、收益账本、经营数据、素材和复购 |
| 后续 V4 | 多平台 C 端小程序 + 商单扩展 | 微信、小红书、抖音小程序消费端共用同一交易底座，并扩展会员经营、商单系统和更多品类 |

## 11. 订单中心

订单不按“活动/商品”粗分，而按履约类型进入消费者和团长的工作台。

| 订单类型 | 核心状态 | 售后重点 |
| --- | --- | --- |
| 活动报名订单 | 待参加、已核销、已取消、退款中 | 场次变更、活动取消、报名信息和保险 |
| 物流订单 | 待发货、已发货、已签收、退货中 | 发货责任、物流轨迹、退换货 |
| 自提订单 | 待自提、已核销、已过期、退款中 | 自提点、提货码、未核销退款 |
| 卡券核销订单 | 待使用、部分使用、已用完、已过期 | 券码、有效期、适用范围、过期争议 |
| 服务预约订单 | 待预约、待到店、已核销、投诉中 | 改期、取消、未服务退款 |

所有订单共用订单详情、售后服务台、退款记录和收益账本，不为不同场景拆出多套交易系统。

## 12. 分销帮卖与收益规则

首发只做一级直接推广。

| 模式 | 规则 | 适用场景 |
| --- | --- | --- |
| 固定佣金分享赚 | 发布方设置固定佣金；消费者价格一致；推广团长成交后获得佣金 | 标准定价活动、普通团购、卡券 |
| 加价帮卖 | 发布方设置结算底价；推广团长设置终端售价；差额扣除平台费后成为推广收益 | 可授权自主定价的供给 |

规则边界：

- 同一供给只能选择固定佣金或加价帮卖。
- 一笔订单只记录一个直接推广团长。
- 优惠券必须记录承担方，推广方发券不得侵蚀发布方底价或造成负收益。
- 全额退款时平台费、发布方收益、推广收益全部取消；部分退款按比例冲回。
- 自动打款上线前，收益账本只表达“待入账、可结算、已结算、售后冻结、退款冲回”等状态。

## 13. 售后服务台

售后服务台是平台区别于普通“群里收钱”的核心。

| 能力 | 首发说明 |
| --- | --- |
| 咨询入口 | 消费者购买前后均可进入咨询 |
| 责任方协同 | 平台客服、发布方客服、必要时推广团长共同处理 |
| 退款工单 | 记录原因、材料、责任方、处理进度 |
| 结算阻断 | 售后未完成时，相关收益保持待入账或冻结 |
| 风险暂停 | 高频退款、投诉或履约异常供给可暂停开团 |

## 14. MVP 范围

第一版必须做：

- 团长注册、社群类型和专属店铺主页。
- 同一微信小程序内的 C 端消费通道和 B 端团长/群主经营通道。
- 新手社群冷启动基础：社群类型选择、新手店铺模板、首单开团清单和任务路径。
- 商品库 / 供给库：V1 保留平台审核货盘和人工推荐，不做完整选品中心。
- 开团发布台：普通团购、卡券团购、活动报名；预售和到店服务可先作为可演示受控能力。
- 详情页、购买须知、分享入口和来源归因。
- 五类订单入口和统一订单详情。
- 售后服务台、退款记录、核销/发货/自提/券码状态。
- 一级固定佣金和一级加价帮卖。
- 收益账本：待入账、可结算、已结算、售后冻结、退款冲回。

第一版不做：

- 多级分销、发展下级、层层转单。
- 保证金账户、经销商层级、会员分层可见。
- 食品生鲜、医疗美容、上门服务、强效果承诺活动。
- 自动选货、自动上架、自动外部发布营销内容。
- V2 选品中心中的热卖榜、转化数据、素材包和经营建议。
- V1 同时建设小红书/抖音小程序消费端或团长 App。
- 按平台拆分多套商品库、多套订单、多套售后或多套结算。
- 未经支付和退款回退核验的真实自动打款承诺。

## 15. 数据对象与接口边界

| 对象 | 最小含义 |
| --- | --- |
| `LeaderAccount` | 团长主体、认证、发布/推广权限、结算状态 |
| `BeginnerLeader` | 零基础或弱基础的新手群主，用于区分冷启动任务和模板推荐 |
| `BusinessChannel` | B 端经营通道，覆盖微信小程序 B 端与后期团长 App |
| `ConsumerChannel` | C 端消费通道，覆盖微信、小红书、抖音小程序消费入口 |
| `ChannelSurface` | 具体端形态，如 `wechat_mini_program`、`xhs_mini_program`、`douyin_mini_program`、`leader_app` |
| `ChannelAttribution` | 记录订单来源平台、分享入口、团长归因和规则版本 |
| `Community` | 社群名称、主类型、兴趣标签、审核状态 |
| `CommunityStarterPath` | 新手社群冷启动路径，包含建店、选品、发布、邀请、首单、复盘 |
| `StarterTemplate` | 按亲子家庭、青年兴趣、创业职场、社区生活、文旅休闲、健康运动、学习成长等方向匹配的新手店铺模板 |
| `Storefront` | 团长专属店铺、模板、货架和分享入口 |
| `SelectionCenter` | V2 选品中心，承接审核货盘、推荐供给、素材包和经营数据 |
| `SupplyItem` | 商品库供给，覆盖活动、实物、自提、卡券、到店服务 |
| `LaunchCampaign` | 开团发布记录，类型为普通团购、预售团购、卡券团购、活动报名、到店服务 |
| `LaunchPolicy` | 售卖时间、库存/名额、售后规则、退款规则、推广模式 |
| `DistributionListing` | 帮卖关系、佣金/底价/终端售价、推广状态 |
| `Attribution` | 分享入口、直接推广团长、锁定时间、规则版本；与 `ChannelAttribution` 共同记录平台入口 |
| `Order` | 消费者、开团、供给、发布方、推广方、实付金额、履约状态 |
| `FulfillmentRecord` | 核销、发货、自提、券码、预约等履约记录 |
| `AfterSalesCase` | 退款、退货、取消、投诉、争议和处理结论 |
| `SettlementLedger` | 平台费、发布方收益、推广收益、优惠承担、退款冲回、结算状态 |

这些对象先进入产品规划表达，不代表本轮已经实现后端或真实接口。`Storefront`、`SupplyItem`、`LaunchCampaign`、`Order`、`AfterSalesCase`、`SettlementLedger` 不按平台拆分，多端只影响入口、展示和归因，不改变交易底座。

## 17. 竞品参考边界

快团团可参考的是“商品库、一键开团、帮卖分销、团购管理、分享转存、订单处理”等产品组织方式；和趣不能照搬的是高复杂多级扩散、强货源平台化、全品类自由开放和未经验证的即时结算承诺。

和趣的差异化切入：

- 从活动、卡券和本地履约切入，而不是只做实物卖货。
- 强调订单服务台和履约监督，而不是只强调开团速度。
- 帮助新手群主从社群冷启动和第一团开始，而不是只服务已有大团长。
- 按社群类型生成店铺、冷启动路径和推荐供给，而不是让团长面对无边界货盘。
- 统一交易底座支撑后续多端承载，而不是每个平台重复建设一套交易系统。
- 首期坚持一级推广和低风险供给白名单。

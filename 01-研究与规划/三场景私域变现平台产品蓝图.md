# 社群店铺驱动的三场景私域变现平台产品蓝图

版本：V2.0

日期：2026-05-27

状态：当前权威产品规划入口

## 1. 一句话定位

和趣帮助拥有社群信任关系的团长或群主，在同一平台小程序中建立与社群类型匹配的专属店铺，从经过审核的货盘中发布或选卖活动、商品和本地服务，完成可追踪、可售后、可结算的私域交易变现。

## 2. 产品主线

`注册团长/社群 -> 选择社群主类型与兴趣标签 -> 生成专属社群店铺模板 -> 发布或选卖审核供给 -> 分享成交 -> 履约/核销/售后 -> 收益结算与复购经营`

| 环节 | 首发能力 | 边界 |
| --- | --- | --- |
| 社群建店 | 一个主类型、多个兴趣标签、专属店铺主页 | 主类型决定页面模板，标签影响推荐货架；不自动上架供给 |
| 统一货盘 | 平台审核供给，团长申请发布或选卖 | 未通过审核不得上架或分享成交 |
| 分享成交 | 店铺、单品与优惠券分享入口，记录唯一直接推广方 | 只支持一级推广，不开放发展下级 |
| 多种履约 | 活动核销、物流配送、商品自提、虚拟权益发放、本地服务到店核销 | 首发采用低风险白名单，不开放高责任品类 |
| 收益售后 | 订阅、平台费、推广收益、优惠承担、退款冲回与工单 | 自动结算上线前需验证支付与回退链路 |

## 3. 首发场景与白名单

首发为三类场景同时提供真实交易入口，但以审核与低风险供给白名单控制开放范围。

| 场景 | 首发供给 | 履约字段 | 暂不开放 |
| --- | --- | --- | --- |
| 活动变现 | 亲子活动、青年兴趣活动、创业交流活动等标准票务/体验 | 类型、票种、场次、地点、名额、履约方、核销、退改规则 | 高风险户外、复杂多日行程、强效果承诺活动 |
| 电商变现 | 低风险标准实物、本地自提商品、电子券/次卡等虚拟权益商品 | 规格、库存、`物流/自提/虚拟权益`交付方式、发货或核销、退换规则 | 食品生鲜、美容医疗、资质或售后责任不清品类 |
| 本地服务变现 | 到店预约核销服务，如摄影、洗护、运动体验、亲子配套服务 | 门店、服务规格、预约时段、服务方、核销、取消与投诉规则 | 上门维修/保洁、医疗美容、强效果或安全风险服务 |

原“付费内容/课程包”不再作为独立首发场景。明确可核销的数字权益可以作为电商虚拟商品进入审核货盘；复杂内容交付、训练营或直播另案验证。

## 4. 角色与责任

| 角色 | 首发能力 | 责任边界 |
| --- | --- | --- |
| 消费者 | 浏览社群店铺、领取优惠、支付、接收交付、申请售后 | 按披露规则核销或提交售后材料 |
| 团长/群主 | 注册社群、使用模板开店、发布供给申请、选卖、分享、查看收益 | 发布自有供给时承担履约责任；推广时披露推荐关系 |
| 供给发布方 | 创建活动、商品或服务，设定推广模式与履约规则 | 对活动完成、商品交付/退换或服务履约负责 |
| 平台运营 | 审核社群与供给、维护白名单、处理争议、暂停风险项目 | 建立规则与处置入口，不无条件代替发布方履约 |
| 支付/合规负责人 | 核验主体、收费、结算、退款回退与平台规则 | 真实自动结算和扩品类前给出可执行结论 |

团长可以同时是供给发布方和推广方；任一消费者订单最多记录一个发布方和一个直接推广方。

## 5. 首发通用底座

| 模块 | 首发能力 | 开发预研重点 |
| --- | --- | --- |
| 社群与店铺 | 社群主类型/标签、店铺模板、主页、订阅状态 | 模板配置、权限与分享入口 |
| 统一供给库 | 发布申请、审核、上下架、货架分类、责任披露 | 共用供给模型和场景/履约扩展 |
| 推广与归因 | 固定佣金分享赚、加价帮卖、优惠券、唯一一级归因 | 互斥收益模式、最低收益校验、优惠归属 |
| 订单与履约 | 下单支付、场景状态、核销/发货/权益发放/到店预约 | 多履约状态机和订单规则快照 |
| 售后与退款 | 工单、取消、退货、未核销/未使用退款、投诉 | 责任分配与退款冲回 |
| 收益与结算 | 订阅、平台费、发布收益、推广收益、优惠承担、结算状态 | 既有结算链路和部分退款复用 |
| 运营治理 | 审核、供给暂停、风险标签、导出与看板 | 白名单、敏感权限、争议处理 |

## 6. 店铺模板与供给经营

### 6.1 社群模板

- 注册时选择一个主类型，如 `亲子家庭`、`青年兴趣`、`创业交流`、`社区生活`；再选择多个兴趣标签。
- 主类型决定店铺首页结构、首屏场景排序和推荐内容提示；标签用于推荐分类和可选货盘筛选。
- 首发模板只辅助展示与选货，不自动替团长发布供给，也不自动对外营销。

### 6.2 发布与选卖

- 平台维护统一审核商品库，供给按 `activity / commerce / local_service` 分类。
- 团长可以申请发布自有供给，经运营审核后上架本人店铺，并可选择是否开放其他团长推广。
- 团长可以把已开放推广的供给加入本人货架；消费者付款前必须看到真实发布/履约主体和推荐关系。
- 风险供给可由运营立即暂停推广或下架，并保留订单售后处理入口。

## 7. 推广、优惠与收入规则

### 7.1 两种一级推广模式

| 模式 | 规则 | 适用供给 |
| --- | --- | --- |
| 固定佣金分享赚 | 发布方设置佣金；消费者价格一致；成交后推广方获得固定佣金 | 统一价格活动、标准商品、统一折扣服务 |
| 加价帮卖 | 发布方设置结算底价；推广方设置终端售价并获得价差空间 | 允许自主定价的活动、商品或服务 |

- 同一个供给只能启用一种推广模式，一笔订单只产生一名直接推广团长收益。
- 首发不开放下级邀请、二级返佣、层层转售；数据对象可保留规则版本和最大深度字段，当前最大深度固定为 `1`。

### 7.2 优惠券

- 首发支持店铺券或单品券，需记录券类型、金额、承担主体、适用范围与使用状态。
- 自售订单的发布方优惠从其应结算金额中承担。
- 推广订单中由推广方发放的券优先从推广收益承担，不得降低发布方底价结算，不得造成推广收益为负。
- 退款时优惠分摊与各方收益按照退款比例同步冲回。

### 7.3 平台收入与退款

| 订单方式 | 发布方应结算 | 推广方收益 | 平台收入 |
| --- | ---: | ---: | ---: |
| 发布方自售 | 消费者实付 - 平台服务费 - 发布方承担优惠 | 0 | 订阅费 + 按实付额计提服务费 |
| 固定佣金推广 | 实付 - 平台费 - 固定佣金 - 发布方承担优惠 | 固定佣金 - 推广方承担优惠 | 订阅费 + 按实付额计提服务费 |
| 加价帮卖 | 供给结算底价 | 实付 - 底价 - 平台费 - 推广方承担优惠 | 订阅费 + 按实付额计提服务费 |

- 全额退款取消或回退发布方结算、推广收益、平台交易服务费和已分摊优惠。
- 部分退款按退款金额占消费者实付金额的比例冲回对应账本项。
- 自动结算是目标能力；真实打款前必须核验现有支付链路是否支持多履约类型、双角色账户、优惠分摊与退款回退。

## 8. 数据对象与接口边界

| 对象 | 最小含义 |
| --- | --- |
| `LeaderAccount` | 团长主体、认证状态、发布/推广权限、结算账户状态 |
| `Community` | 社群、主类型、兴趣标签、所属团长、审核状态 |
| `Storefront` | 专属店铺、模板、货架配置、订阅与分享入口 |
| `Offering` | `activity / commerce / local_service` 供给、发布方、推广模式、审核与责任规则 |
| `FulfillmentVariant` | `session_ticket / logistics / pickup / virtual_voucher / store_booking` 交付类型 |
| `DistributionListing` | 推广团长、佣金或终端定价、优惠约束、上架状态 |
| `Attribution` | 分享入口、直接推广方、锁定时间、规则版本、最大层级 `1` |
| `Order` | 消费者、供给、发布方、推广方、实付额、交付与退款状态 |
| `SettlementLedger` | 平台费、发布结算、推广收益、优惠承担、退款冲回和结算状态 |
| `SubscriptionPlan` | 店铺经营套餐、周期、可用能力和订阅状态 |
| `AfterSalesCase` | 取消、退款、退货、核销争议或服务投诉及处理结论 |

## 9. 阶段路线与任务

| 阶段 | 产品交付 | 开发与支付任务 | 运营任务 | 门槛 |
| --- | --- | --- | --- | --- |
| A：主线与规格固化 | 当前蓝图、同步材料、三场景原型 | 模型、状态机、收益和优惠规则评审 | 首发白名单草案 | 口径一致 |
| B：通用底座预开发 | 社群建店、模板、供给、交易流程规格 | 归因、订单、履约、售后、结算复用评估 | 社群类型、供给责任与团长名单 | 可排期可验证 |
| C：受控真实开放 | 三场景真实入口与运营规则 | 联调通过的交易链和风险暂停能力 | 审核供给、处理售后、追踪经营数据 | 支付/履约/合规通过 |
| D：成交效率扩展 | AI 营销话术与群接龙另案验证 | 只针对已验证需求建设 | 收集复购与营销效率数据 | 三场景数据成立 |

## 10. 暂不纳入首发

- 多级返佣、发展下级和层层加价。
- 自动为团长上货或自动向外部平台发布营销内容。
- AI 朋友圈/社群/小红书/抖音话术生成与群接龙，均进入二期验证。
- 食品生鲜、医疗美容、上门服务、复杂内容课程、直播训练营与广告变现。
- 支付与退款回退核验完成前的真实自动打款承诺。

## 11. 上线前必须复核

- 既有微信支付/结算能力是否支持发布方、推广方、平台费、优惠承担及退款回退并存。
- 活动、电商与本地服务的经营主体、履约披露、售后责任和平台纠纷处理规则。
- 优惠券与推广收益在全额/部分退款场景中的账务一致性。
- 平台规则、消费者保护、禁止或限制品类、营销内容审核与运营处置流程。

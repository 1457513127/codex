# 和趣当前产品策划

版本：V6.0
更新日期：2026-06-02
状态：当前唯一产品口径

## 1. 产品定位

和趣是面向私域社群经营者的社群变现平台，服务两类核心客户：

- 已有社群经营者：已经有微信群、客户关系、兴趣圈层或本地触达，需要把发品、下单、履约、售后、收益解释变得可管理。
- 零基础新手群主：还没有成熟社群和供给，需要明确服务谁、怎么建社群、怎么开第一团、怎么完成第一笔真实交易。

和趣不是单一商城、单点活动报名工具或纯分销平台。它的核心价值是用“社群经营者 + 可信供给 + 消费者订单 + 平台运营监督”组织交易，把群聊里的口头信任交易变成有规则、有凭证、有售后、有收益解释的交易闭环。

## 2. 赛道与市场口径

和趣进入三层赛道：

| 层级 | 口径 | 说明 |
| --- | --- | --- |
| 主赛道 | 私域社群变现 / 社群团购 | 团长、群主、主理人通过私域社群和小程序完成交易转化 |
| 延展赛道 | 本地生活交易 | 活动报名、卡券核销、到店服务等线下履约交易 |
| 首期切口 | 社群冷启动、活动报名、卡券团购、低风险普通团购 | 先验证低风险真实交易闭环，不做全品类大平台 |

市场空间使用 TAM / SAM / SOM 表达：

- TAM：私域电商和本地生活交易大盘。
- SAM：微信小程序、社群经营、团购发布、卡券核销、本地活动/服务交易。
- SOM：首期可触达城市、团长、新手群主、供给和真实订单。

公开市场数据只说明方向值得验证，不代表和趣已拥有收入、市场份额或确定付费率。SOM 后续按 `可触达团长数 × 月均有效开团数 × 单团支付订单数 × 客单价 × 平台可确认收入率` 滚动测算。

## 3. 用户与角色

| 角色 | 核心诉求 | 平台要解决 |
| --- | --- | --- |
| 已有社群经营者 | 从 1 到 N 放大交易 | 发品、开团、订单、履约、售后、收益解释 |
| 零基础新手群主 | 从 0 到 1 完成第一团 | 选方向、建社群、搭主页、选首批供给、首单转化 |
| 供给方 | 把活动、商品、卡券、服务卖出去 | 供给入库、审核、履约规则、帮卖授权 |
| 推广团长 | 用自己的社群触达帮卖 | 分享归因、固定佣金、加价帮卖、推广收益 |
| 消费者/团员 | 安全下单并清楚获得什么 | 购买须知、订单、核销/收货/到店、售后退款入口 |
| 平台运营 | 控制供给和交易风险 | 审核、白名单、风险暂停、售后协同、账本监督 |

## 4. 产品主线

`选择经营方向 -> 社群冷启动 -> 建店 -> 供给入库/选品 -> 开团发布 -> 分享成交 -> 履约售后 -> 收益结算 -> 复购经营 -> 选品中心放大`

当前策划方法是“先边界、再场景、再版本、再功能”：不在产品未定稿前把商品库、订单、分销等功能写成最终规格。

## 5. 变现场景优先级

| 场景 | 当前判断 | 阶段边界 |
| --- | --- | --- |
| 活动变现 | P0 首发主验证 | 优先亲子家庭、青年兴趣、创业职场、社区生活等低风险白名单活动 |
| 卡券变现 | P0 首发标准化交易 | 电子券、次卡、体验券、权益包，用于验证凭证、有效期、核销、复购 |
| 低风险电商 | P1 对照验证 | 标准商品、自提商品、低复杂履约，不做食品生鲜和强售后品类 |
| 本地服务 | P1/P2 受控后置 | 先做到店预约/核销，规模化门店服务、上门服务和复杂改期投诉后置 |
| 内容/课程 | 暂不独立首发 | 可作为体验课、工作坊、活动或卡券供给；系统课、讲师分账、强效果承诺后置 |

活动货盘分层：

| 层级 | 类型 | 示例 |
| --- | --- | --- |
| 首发白名单 | 亲子家庭 | 亲子手作、研学体验、儿童剧/展览、亲子运动、营地轻体验 |
| 首发白名单 | 青年兴趣 | Citywalk、桌游社交、摄影采风、手作体验、音乐/观影、轻运动 |
| 首发白名单 | 创业职场 | 创业沙龙、资源对接、项目路演、行业交流、技能工作坊 |
| 首发白名单 | 社区生活 | 邻里集市、社区体验课、便民活动、节日主题活动 |
| 受控扩展 | 文旅休闲、健康运动、学习成长 | 周边短途、非遗体验、瑜伽、羽毛球、读书会、公开课 |

暂不开放：高风险户外、多日复杂行程、强效果承诺培训、医疗健康承诺、金融投资承诺、未成年人高风险托管活动。

## 6. MVP 范围

V1 必须做：

- 社群建店：团长注册、社群类型、兴趣标签、专属社群主页。
- 新手冷启动：新手模板、首单开团清单、建店/选品/发布/邀请/首单/复盘任务路径。
- 供给库基础：平台审核货盘、人工推荐、供给责任披露。
- 开团发布：活动报名、卡券团购、低风险普通团购；预售和到店服务先受控验证。
- 交易闭环：详情页、购买须知、分享归因、订单详情、核销/发货/自提/券码/预约状态。
- 售后服务台：咨询、退款工单、责任方协同、结算阻断、风险暂停。
- 收益账本：待入账、可结算、已结算、售后冻结、退款冲回。
- 一级推广：固定佣金分享赚、加价帮卖二选一。

V1 不做：

- 多级分销、发展下级、层层转单。
- 保证金账户、经销商层级、会员分层可见。
- 强监管或高售后品类：食品生鲜、医疗美容、上门服务、高风险活动。
- 自动选货、自动上架、自动外部发布营销内容。
- 小红书/抖音消费端、团长 App、复杂商单、独立内容课程体系。
- 支付链路未核验前的真实自动分账或即时打款承诺。

## 7. 功能沉淀

当前功能只做策划沉淀，不写死最终规格：

| 功能域 | 当前口径 |
| --- | --- |
| 商品库/供给库 | 统一管理活动票务、实物商品、自提商品、虚拟卡券、到店服务；所有供给上线前审核 |
| 开团发布 | 采用“供给库 + 开团类型”，不再用割裂的创建活动/创建商品长表单作为主入口 |
| 订单中心 | 按履约类型管理：活动报名、物流、自提、卡券核销、服务预约 |
| 分销帮卖 | 首发只做一级固定佣金和一级加价帮卖；一笔订单只记录一个直接推广方 |
| 优惠券 | 首发可保留，但必须明确承担方，不能侵蚀发布方底价或造成推广方负收益 |
| 选品中心 | V2 能力，先人工审核和推荐；后续再做热卖榜、素材包、推荐理由和经营建议 |
| 多端承载 | V1 同一微信小程序 C/B 双通道；后续团长 App、小红书/抖音消费端共用统一交易底座 |

## 8. 多期路线

| 阶段 | 核心目标 | 进入下一阶段条件 |
| --- | --- | --- |
| V0 人工验证 | 用人工货盘、表格台账、轻量下单验证真实订单、履约、退款和团长复盘意愿 | 形成首批供给规则、团长路径、订单台账、售后处理方式和 V1 必做/不做边界 |
| V1 交易闭环 | 同一微信小程序跑通 C 端下单和 B 端经营，帮助新手完成第一团 | 至少一个首发场景形成可复用开团模板，售后和收益解释能被平台规则承接 |
| V2 供给放大 | 建设选品中心、审核货盘分层、推荐供给包、素材包，受控扩展低风险商品/服务 | 审核货盘和推荐逻辑可被多社群复用，新增场景未显著拉高售后风险 |
| V3 经营效率 | 团长 App 或专业 B 端工具、经营数据、复购、素材和售后批处理 | 高频团长出现明确效率瓶颈，且有足够样本愿意迁移 |
| V4 多端与复杂交易 | 多平台 C 端入口、复杂商单、内容/课程受控验证 | 统一交易底座稳定，多端带来新增订单且不拆散货盘、订单、售后和结算 |

版本按验证门槛推进，不按日历强推。

## 9. 竞品参考边界

| 参考对象 | 可借鉴 | 不照搬 |
| --- | --- | --- |
| 快团团 | 商品库、开团、帮卖、订单管理 | 全品类开放、复杂裂变、即时结算承诺 |
| 群接龙 | 社群接龙、团购发布、供应商/品牌入驻 | 把接龙作为唯一形态，弱化履约凭证和售后 |
| 有赞 | 分销市场、团长招募、业绩统计、小程序交易 | 面向成熟商家的重后台和复杂商家体系 |
| 本地生活平台 | 活动、卡券、到店服务、内容带交易 | 公域流量投放、大规模商户入驻、达人体系 |

和趣差异：从活动、卡券和本地履约切入，强调订单服务台、履约监督、新手冷启动、低风险白名单和收益解释，而不是只做选品/分销。

## 10. 当前待确认

- 首发城市、首批团长样本、首批供给数量和真实订单门槛。
- 活动、卡券、普通团购进入 V2 的复用指标。
- 平台运营在售后纠纷中的介入深度和责任边界。
- 支付主体、分账、退款回退和结算阻断能力是否可复用。
- 何时从“功能方案沉淀区”拆出商品库、订单、分销、结算等独立 PRD。

# 和趣项目上下文总览

版本：V4.2

更新日期：2026-05-27

用途：新会话、跨部门沟通和项目交接的第一阅读入口

## 1. 当前一句话定位

> 和趣是帮助已有社群经营者和零基础新手群主完成社群生意从 0 到 1、从 1 到 N 的社群团购与本地交易变现平台：V1 在同一微信小程序内承载 C 端消费通道和 B 端团长/群主经营通道，通过社群冷启动、社群建店、商品库/供给库、开团发布、分销帮卖、订单履约、售后退款和收益结算，把微信群里的信任交易变成可管理、可追踪、可复购的经营闭环；后续扩展团长 App 和微信/小红书/抖音小程序消费端。

当前执行基线以 [和趣社群团购与本地交易变现平台产品蓝图](./01-研究与规划/三场景私域变现平台产品蓝图.md) 为准。

## 2. 当前产品主线

`选择经营方向 -> 社群冷启动 -> 建店 -> 商品库/供给库 -> 开团发布 -> 分享成交 -> 履约售后 -> 收益结算 -> 复购经营 -> 选品中心放大`

新手群主是核心客户之一，但首发“起号”只定义为社群冷启动，不做小红书、抖音或视频号公域账号代运营。选品中心是 V2 增强版能力，V1 只保留审核货盘和人工推荐。

产品形态口径：V1 是同一微信小程序 C/B 双通道，不拆两个小程序；后续团长 App 是 B 端专业经营工具，微信、小红书、抖音小程序是 C 端消费入口，均共用商品库、开团、订单、售后、归因和收益账本。

本轮已将发布模式正式调整为“商品库/供给库 + 开团类型”。开团类型包括：

- 普通团购。
- 预售团购。
- 卡券团购。
- 活动报名。
- 到店服务。

## 3. 已经发生的产品阶段变化

| 阶段 | 原判断 | 当前结论 |
| --- | --- | --- |
| 早期概念 | 全品类私域社群变现 SaaS | 范围过大，不作为 MVP |
| 活动验证 | 亲子活动交易留痕与售后 | 是首期最接近现有基础的验证入口 |
| 三场景规划 | 活动、电商、本地服务 | 进一步重组为五种开团/履约形态 |
| 商单校准 | 关注供给和分销 | 收敛为订单履约、售后和结算监督 |
| 首次全员立项会 | 内部同步版材料 | 改为完整介绍赛道、市场、用户、功能、路线和边界 |
| 新手群主补充 | 只服务已有私域团长 | 零基础群主成为核心客户之一，V1 做社群冷启动，V2 做选品中心 |
| 产品形态补充 | 只表达一个微信小程序 | 升级为 V1 微信小程序 C/B 双通道，后续 B 端 App + 多平台 C 端小程序 |

## 4. 市场与证据口径

市场空间采用 TAM / SAM / SOM：

- TAM：私域电商和本地生活交易大盘。
- SAM：微信小程序、社群团购、卡券核销、本地活动/服务交易。
- SOM：首期可触达城市、已有团长、新手群主、供给和真实订单。

外部数据只用于说明方向值得验证，不证明和趣已拥有收入或市场份额。

可使用的数据口径：

- 艾瑞咨询《2025年中国私域电商行业趋势白皮书》：私域电商 2024 年约 2.3 万亿元，2025 年预计突破 3 万亿元，2027 年用户规模预计 9.4 亿。
- QuestMobile 2025 秋季报告：2025 年 8 月微信小程序端整体流量约 9.50 亿。
- 艾瑞转引本地服务市场口径：2025 年中国本地服务市场规模预计约 35.3 万亿元，线上本地服务约 10.9 万亿元。

## 5. 当前应做

- 建设社群建店、商品库/供给库和开团发布台。
- 增加新手社群冷启动基础：社群类型选择、新手店铺模板、首单开团清单和任务路径。
- 在同一微信小程序中承载 C 端消费通道和 B 端团长/群主经营通道。
- 首期重点验证活动报名、普通团购、卡券团购。
- 订单中心按履约类型拆清：活动报名、物流、自提、卡券核销、服务预约。
- 建设售后服务台，支撑咨询、退款工单、责任方协同和结算阻断。
- 建设收益账本，记录待入账、可结算、已结算、售后冻结和退款冲回。
- 保留一级固定佣金分享赚和一级加价帮卖。

## 6. 当前不做

- 多级分销、发展下级、层层转单。
- 保证金账户、经销商层级、会员分层可见。
- 食品生鲜、医疗美容、上门服务、高风险户外和强效果承诺活动。
- 自动选货、自动上架、自动外部发布营销内容。
- V2 选品中心中的热卖榜、转化数据、素材包和经营建议。
- V1 同时建设小红书/抖音小程序消费端或团长 App。
- 按平台拆分多套商品库、多套订单、多套售后和多套结算。
- 支付链路未核验前的真实自动打款承诺。

## 7. 本轮项目会要形成的全员共识

| 议题 | 必须形成的理解 |
| --- | --- |
| 项目是什么 | 帮助已有社群经营者和零基础新手群主做社群生意的平台，不是单点活动工具 |
| 在什么赛道 | 私域电商 / 社群团购为主，本地生活交易为延展 |
| 为什么值得做 | 公域获客贵、私域经营需求提升、小程序交易入口成熟 |
| 怎么发布 | 商品库/供给库 + 开团类型，而不是割裂长表单 |
| 怎么成交 | 社群店铺、单品页、团购页、卡券和分享链接 |
| 怎么承载 | V1 同一微信小程序双通道；后续 B 端 App + 多平台 C 端小程序 |
| 怎么分销 | 一级固定佣金或一级加价帮卖 |
| 怎么履约 | 活动核销、物流、自提、卡券核销、服务预约 |
| 怎么售后 | 订单服务台和退款工单 |
| 怎么结算 | 收益账本先解释状态，自动打款需后续核验 |
| 怎么迭代 | V1 微信小程序双通道 + 交易闭环 + 新手冷启动，V2 选品中心 + 履约扩展，V3 团长 App + 经营效率，V4 多平台 C 端小程序 + 商单扩展 |

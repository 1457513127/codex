# 和趣亲子活动平台项目上下文总览

版本：V3.0
更新日期：2026-05-27
用途：新会话、跨部门沟通和项目交接的第一阅读入口

## 1. 当前一句话定位

> 和趣面向团长与群主建设社群店铺驱动的私域变现平台：注册社群并选择类型后生成专属店铺，从审核货盘中发布或选卖活动、电商商品与到店本地服务，通过一级推广、优惠券、履约售后与结算完成经营闭环。

当前执行基线以 [社群店铺驱动的三场景私域变现平台产品蓝图](./01-研究与规划/三场景私域变现平台产品蓝图.md) 为准。小红书调研只作为需求线索和优先级依据：活动变现为主验证场景，电商变现为对照验证场景，本地服务为受控补充场景。此前低风险活动验证方案，以及活动/商品/付费内容版图，均保留为历史演进依据，不再代表当前首发定义。

## 2. 已经发生的产品阶段变化

| 阶段 | 原判断 | 当前结论 |
| --- | --- | --- |
| 早期概念 | 全品类私域社群变现 SaaS | 范围过大，不能作为 MVP |
| 场景收缩 | 成人低风险线下体验交易留痕 | 被真实业务修正 |
| 交易验证 | 亲子活动发布、报名、保险、核销、退款留痕 | 已有产品与业务验证基础 |
| 当前功能蓝图 | 场景与经营主线如何成立 | 社群类型建店、审核货盘、活动/电商/到店本地服务三场景受控首发 |
| 小红书调研校准 | 五场景需求线索比较 | 活动主验证、电商对照、本地服务受控补充；会员订阅和知识/内容服务后置 |

## 3. 不可混淆的角色定义

| 角色 | 当前定义 | 不应被写成 |
| --- | --- | --- |
| 公司/平台 | 精选采购或代售、承接交易与家长售后、做审核和数据治理 | 无条件担保方 |
| 供给发布团长 | 发布活动、商品或本地服务，并承担对应履约/发货/到店服务责任 | 推广者的履约兜底方 |
| 推广团长 | 选择允许推广的供给，通过固定佣金或加价帮卖进行一次直接推广 | 二级分销上级或隐藏履约责任方 |
| 消费者/家长 | 通过店铺或分享入口购买、接收交付并申请售后 | - |

## 5. 当前应做和不应做

### 当前应做

- 建设社群类型/标签、专属店铺模板、审核供给库、一级归因、订单、收益、优惠券、售后和运营治理底座规格。
- 以活动变现作为主验证场景，优先完善场次、票种、报名、核销、退改和经营复盘。
- 将电商变现作为对照验证场景，明确物流、自提、虚拟权益三种履约方式。
- 将到店本地服务作为受控补充场景，仅保留预约、到店核销、取消投诉等低风险能力。
- 复核已有结算链路对双角色、两种推广模式、优惠分摊和按比例退款回退的支持程度。
- 让运营分别建立三场景低风险白名单、社群类型模板、责任材料和目标团长名单。

### 当前不做

- 多级分销、邀请下级赚佣金、层层转单。
- 上门服务、食品生鲜、医疗美容、复杂内容交付、广告变现、直播训练营和群内重互动。
- AI 多平台营销话术和群接龙作为首发建设项。
- 把儿童信息、保险信息用于推广画像。
- 未完成既有支付链路复用核验前宣称真实自动结算可以上线。

## 6. 本轮项目会要同步的产品路线

| 议题 | 会议需形成的产品共识 |
| --- | --- |
| 产品主线 | 社群类型建店、审核货盘、三场景履约与一级收益闭环的统一理解 |
| 场景优先级 | 活动主验证、电商对照验证、本地服务受控补充 |
| 场景白名单 | 活动、电商和到店服务首发允许/限制品类及责任披露 |
| 店铺与货盘 | 模板分类、发布/选卖审核和分享归因规则 |
| 收益与优惠 | 固定佣金/加价互斥、优惠承担、退款冲回及最低收益保护 |
| 迭代路线 | 首发交易闭环、场景验证、经营效率增强、后续扩品类 |
| 证据边界 | 小红书数据是需求线索，不是营收、市场规模或 ROI 证明 |

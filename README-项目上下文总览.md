# 和趣亲子活动平台项目上下文总览

版本：V1.0  
更新日期：2026-05-26  
用途：新会话、跨部门沟通和项目交接的第一阅读入口

## 1. 当前一句话定位

> 和趣面向亲子微信群场景，承接公司精选亲子活动的交易与售后，并让经过审核的团长通过轻主页进行一级推荐分销；主办方负责实际履约，平台负责交易记录和售后承接。

## 2. 已经发生的产品阶段变化

| 阶段 | 原判断 | 当前结论 |
| --- | --- | --- |
| 早期概念 | 全品类私域社群变现 SaaS | 范围过大，不能作为 MVP |
| 场景收缩 | 成人低风险线下体验交易留痕 | 被真实业务修正 |
| 交易验证 | 亲子活动发布、报名、保险、核销、退款留痕 | 已有产品与业务验证基础 |
| 当前增长阶段 | 是否加入分销、社群主页和扩供给 | 先验证轻主页 + 一级帮卖 + 精选亲子供给，不同时开放商品 |

## 3. 不可混淆的角色定义

| 角色 | 当前定义 | 不应被写成 |
| --- | --- | --- |
| 公司/平台 | 精选采购或代售、承接交易与家长售后、做审核和数据治理 | 无条件担保方 |
| 主办方 | 实际提供活动和现场履约的一方 | 推荐团长 |
| 团长 | 有亲子微信群的一级推荐者，分享允许推广的活动并获得佣金 | 活动履约方或二级分销上级 |
| 家长 | 代儿童报名、确认规则、支付和申请售后 | - |

## 5. 当前应做和不应做

### 当前应做

- 明确 `主办方`、`团长`、平台的展示与责任边界。
- 建设团长轻主页：身份、推荐关系、在售活动货架、信任说明和分享归因。
- 建设一级帮卖：推广订单、佣金状态、履约完成并过退款窗口后可结算。
- 补数据埋点和经营口径，用真实试点判断增量、退款质量和贡献毛利。
- 对公司自营和精选外部亲子活动建立分销准入与暂停规则。

### 当前不做

- 商品团购、实物供应链、物流与退换货。
- 多级分销、邀请下级赚佣金、裂变排行。
- 内容社区、群成员互动或资料库。
- 把儿童信息、保险信息用于推广画像。
- 未确认支付/法务路径前自动结算个人团长佣金。

## 6. 本轮项目会要做出的决定

| 议题 | 会议需形成的决定 |
| --- | --- |
| 可行性 | 是否同意以受控一级帮卖进入 0-3 个月验证 |
| 角色 | 公司、主办方、团长、家长的责任文案和运营处理方式 |
| 供给 | 首批允许分销活动白名单与禁止/限制类型 |
| 数据 | 收入/退款口径负责人，新增归因、佣金、成本、退款原因字段 |
| 资金 | 团长结算主体、延迟结算和退款后处理方案的确认任务 |
| 研发 | MVP 页面及接口范围、粗评估和依赖 |
| 运营 | 首批团长、活动、内容披露与售后 SOP |
| 试点门槛 | 是否接受试点人数、活动数、质量护栏和停止条件 |

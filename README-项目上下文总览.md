# 和趣亲子活动平台项目上下文总览

版本：V2.0
更新日期：2026-05-27
用途：新会话、跨部门沟通和项目交接的第一阅读入口

## 1. 当前一句话定位

> 和趣面向经营型团长建设私域变现平台，首轮同步规划活动、电商商品与付费内容/课程包三类交易能力；团长可自营供给或进行一级加价帮卖，平台通过工具订阅和按终端成交额抽佣获得收入。

当前执行基线以 [三场景私域变现平台产品蓝图](./01-研究与规划/三场景私域变现平台产品蓝图.md) 为准。此前仅围绕低风险活动/服务试单的方案保留为风险收敛依据，不再代表当前功能版图。

## 2. 已经发生的产品阶段变化

| 阶段 | 原判断 | 当前结论 |
| --- | --- | --- |
| 早期概念 | 全品类私域社群变现 SaaS | 范围过大，不能作为 MVP |
| 场景收缩 | 成人低风险线下体验交易留痕 | 被真实业务修正 |
| 交易验证 | 亲子活动发布、报名、保险、核销、退款留痕 | 已有产品与业务验证基础 |
| 当前功能蓝图 | 是否规划多场景私域变现 | 活动、商品、付费内容/课程包同步规划，模块并行预研，按可行性门槛分期开放 |

## 3. 不可混淆的角色定义

| 角色 | 当前定义 | 不应被写成 |
| --- | --- | --- |
| 公司/平台 | 精选采购或代售、承接交易与家长售后、做审核和数据治理 | 无条件担保方 |
| 供给发布团长 | 发布活动、商品或付费内容，并承担对应履约/发货/交付责任 | 推广者的履约兜底方 |
| 推广团长 | 选择允许帮卖的供给，设置终端售价并进行一次直接推广 | 二级分销上级或隐藏履约责任方 |
| 消费者/家长 | 通过店铺或分享入口购买、接收交付并申请售后 | - |

必须记住的限制：

- 当前表格没有团长来源、佣金、入口访问、转化、采购成本或复购字段，不能用来宣称帮卖 ROI 已成立。
- 有 `42` 场活动出现累计退款金额高于实际收入字段，正式经营核算前必须完成字段口径对账。
- 名称启发式分类显示户外/研学/采摘类退款风险偏高，首批分销货架需人工限制。

## 5. 当前应做和不应做

### 当前应做

- 建设三场景共用的账号、店铺、供给、一级归因、订单、收益、订阅、售后和运营治理底座规格。
- 为活动、商品、付费内容/课程包并行完成模块拆分与页面流程设计。
- 复核已有结算链路对双角色、终端额抽佣和按比例退款回退的支持程度。
- 让运营分别摸排三类供给来源、经营型团长需求、履约责任与售后现实成本。

### 当前不做

- 多级分销、邀请下级赚佣金、层层转单。
- 本地服务、广告变现、直播训练营和群内重互动。
- 把儿童信息、保险信息用于推广画像。
- 未完成既有支付链路复用核验前宣称真实自动结算可以上线。

## 6. 本轮项目会要做出的决定

| 议题 | 会议需形成的决定 |
| --- | --- |
| 可行性 | 是否同意以受控一级帮卖进入 0-3 个月验证 |
| 角色 | 公司、主办方、团长、家长的责任文案和运营处理方式 |
| 供给 | 首批允许分销活动白名单与禁止/限制类型 |
| 数据 | 收入/退款口径负责人，新增归因、佣金、成本、退款原因字段 |
| 资金 | 团长结算主体、延迟结算和退款后处理方案的确认任务 |
| 研发 | MVP 页面及接口范围、粗评估和依赖 |
| 运营 | 首批团长、活动、内容披露与售后 SOP |
| 试点门槛 | 是否接受试点人数、活动数、质量护栏和停止条件 |

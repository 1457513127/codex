# 专题索引

本文件帮助新对话快速找到对应专题的权威材料和沉淀位置。

## 1. 当前权威入口

| 专题 | 主要文件 | 说明 |
| --- | --- | --- |
| 项目总览 | `README-项目上下文总览.md` | 当前定位、主线、边界和共识 |
| 稳定记忆 | `docs/PROJECT_MEMORY.md` | 跨会话必须知道的稳定结论 |
| 产品蓝图 | `01-研究与规划/三场景私域变现平台产品蓝图.md` | 当前权威产品规划 |
| 项目会材料 | `02-项目会/三场景产品蓝图同步会主讲材料.md` | 首次全员介绍主讲材料 |
| PPT | `02-项目会/和趣社群店铺三场景产品蓝图同步会.pptx` | 当前项目会演示稿 |
| 原型 | `03-演示界面/和趣项目会演示原型.html` | 可点击演示原型 |
| 走查脚本 | `03-演示界面/演示界面走查脚本.md` | 原型讲解路径 |

## 2. 产品专题

| 专题 | 当前材料 | 沉淀建议 |
| --- | --- | --- |
| 产品定位与商业模式 | 产品蓝图、上下文总览 | 决策进入 `决策记录.md` |
| 用户分层与新手冷启动 | 产品蓝图、主讲材料 | 稳定规则进入 `PROJECT_MEMORY.md` |
| 活动变现与货盘 | 产品蓝图、上下文总览 | 新品类先进入待确认问题或专题文档 |
| 商品库与开团发布 | 产品蓝图、原型、主讲材料 | 后续 PRD 可独立成文 |
| 分销帮卖与收益规则 | 产品蓝图、主讲材料 | 涉及结算的变更必须进决策记录 |
| 订单售后与结算 | 产品蓝图、原型 | 支付核验结果单独沉淀 |
| 多端承载与 App 路线 | 产品蓝图、主讲材料 | 只作为承载路线，不覆盖业务主题 |
| 市场研究与竞品 | 研究目录、主讲材料 | 来源和证据边界必须写清 |
| PPT 与会议材料 | 项目会目录 | 只有版本口径明确后再更新 PPT |
| 原型与界面走查 | 演示界面目录、设计系统 | 所有界面复用设计 tokens |

## 3. 历史资料

| 目录 | 用途 |
| --- | --- |
| `00-原始资料/` | 原始 PDF、截图、数据表，不直接作为当前结论 |
| `01-研究与规划/` | 产品研究、历史规划、当前蓝图 |
| `02-项目会/` | 项目会材料、执行包和 PPT |
| `03-演示界面/` | 可点击原型和演示脚本 |
| `docs/` | 记忆、变更政策和更新日志 |

## 4. 新专题文档命名建议

新专题文档建议放在 `01-研究与规划/` 或 `02-项目会/`，命名格式：

`主题-用途.md`

示例：

- `活动变现货盘与审核规则.md`
- `商品库与开团发布PRD.md`
- `订单售后与收益账本PRD.md`
- `团长App阶段规划.md`

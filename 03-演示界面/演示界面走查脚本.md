# 和趣首次全员产品立项会演示原型走查脚本

版本：V4.0

日期：2026-05-27

## 走查目的

本原型用于第一次全员理解项目：和趣不是单点活动工具，也不是普通商城，而是面向团长、群主和私域经营者的社群团购与本地交易变现平台。

主线固定为：

`社群建店 -> 商品库/供给库 -> 开团发布 -> 社群分享/分销帮卖 -> 下单支付 -> 履约核销/发货/自提/券码 -> 售后退款 -> 收益结算 -> 复购经营`

## 路径一：先讲项目是什么

1. 打开 `项目总览`。
2. 说明和趣进入的是私域电商、社群团购和本地生活交易交叉赛道。
3. 强调核心能力不是“建店”，而是“商品库 + 开团发布 + 分销帮卖 + 订单履约 + 售后结算”。

验收点：

- 页面能表达 5 种开团类型、5 种订单履约和一级分销。
- 不再把项目讲成单一亲子活动工具。

## 路径二：讲市场与赛道

1. 点击 `市场与赛道`。
2. 讲 TAM / SAM / SOM：
   - TAM：私域电商和本地服务大盘。
   - SAM：微信小程序、社群团购、卡券核销、本地活动/服务交易。
   - SOM：首期可触达团长、供给和真实订单。
3. 强调市场数据只证明方向值得验证，不证明已获得收入。

验收点：

- 市场页必须出现私域电商、小程序流量、本地服务三个信号。
- 不宣称已有市场份额或 ROI。

## 路径三：从社群店铺进入商品库

1. 点击 `团长店铺`。
2. 说明团长在同一小程序内拥有专属店铺主页。
3. 点击 `商品库`，展示活动票务、实物、自提、虚拟卡券、到店服务五类供给。

验收点：

- 商品库不是纯商品列表，而是统一供给库。
- 所有供给发布前需要明确审核、履约责任和售后规则。

## 路径四：重点讲发布开团改造

1. 点击 `发布开团`。
2. 说明发布入口从两张长表单改为“从商品库选 / 新建供给入库 / 预览页面”。
3. 讲五种开团类型：
   - 普通团购。
   - 预售团购。
   - 卡券团购。
   - 活动报名。
   - 到店服务。
4. 强调这参考快团团的商品库和开团组织方式，但不照搬其全品类自由开放和复杂扩散。

验收点：

- 发布页不能再表现成割裂的“创建活动”和“创建商品”。
- 预售只表达先收单后履约，不承诺复杂成团和资金监管。

## 路径五：讲三类首期交易形态

1. 点击 `活动报名`，讲亲子、青年、创业活动如何配置场次、票种、报名、保险、核销和退款。
2. 点击 `普通/预售`，讲普通团购与预售团购的字段差异。
3. 点击 `卡券团购`，讲电子券、次卡、权益包的有效期、核销和退款规则。

验收点：

- 活动报名、普通团购、卡券团购是首期重点。
- 预售和到店服务是第二期履约复杂度验证，不抢首发主线。

## 路径六：讲订单中心

1. 点击 `订单中心`。
2. 说明订单按履约类型管理：
   - 活动报名订单。
   - 物流订单。
   - 自提订单。
   - 卡券核销订单。
   - 服务预约订单。
3. 强调所有订单共用详情、售后、退款和收益账本。

验收点：

- 订单页必须能看出不同履约类型。
- 不为活动、商品、卡券、服务拆多套交易系统。

## 路径七：讲分销帮卖

1. 点击 `分销帮卖`。
2. 讲固定佣金分享赚和加价帮卖。
3. 说明一单只有一个直接推广团长。
4. 强调不做多级分销、发展下级、层层转单。

验收点：

- 分销能解释团长为什么愿意分享。
- 也能解释退款后收益如何冲回。

## 路径八：讲售后和收益

1. 点击 `售后服务台`。
2. 讲平台客服、发布方、退款工单和结算阻断。
3. 点击 `收益账本`。
4. 讲待入账、可结算、已结算、售后冻结、退款冲回。

验收点：

- 售后服务台是平台护城河，而不是大群社区。
- 真实自动打款前，只承诺账本状态和规则，不承诺即时到账。

## 路径九：讲迭代路线

1. 点击 `迭代路线`。
2. 讲：
   - V1：交易闭环。
   - V2：履约扩展。
   - V3：经营效率。
   - V4：商单扩展。
3. 强调 AI 文案、群接龙、会员经营、复杂商单均后置。

验收点：

- 全员能理解先做什么、后做什么、为什么不一次性全做。
- 不把多级分销、保证金、强监管品类放进首发承诺。

## 最终验收清单

- 原型能从 `商品库 -> 发布开团 -> 订单中心 -> 售后服务台 -> 收益账本` 走通。
- 原型展示普通团购、预售团购、卡券团购、活动报名和到店服务的差异。
- 原型不出现多级分销、保证金账户、经销商层级作为首发入口。
- 所有页面继续引用 `design-system/airbnb-tokens.css`。

## 1. 演示定位

本原型用于同步一条可以交给开发和运营分工的首发主线：团长按社群类型建店，从审核货盘发布或选卖活动、电商与到店本地服务，消费者完成交易与履约，平台跟踪售后和一级收益。

会议中统一强调：

- 三类首发交易入口为 `活动`、`电商`、`本地服务`，均以低风险白名单与审核控制范围。
- 电商履约同时展示 `物流`、`自提`、`虚拟权益`；本地服务仅展示 `到店预约核销`。
- 推广支持 `固定佣金分享赚` 或 `加价帮卖`，两者互斥且只允许一个直接推广团长。
- 优惠券进入首发；AI 营销话术和群接龙进入二期；真实自动结算须先核验支付链路。

## 2. 主讲路径

### 路径 1：产品总览

操作：选择左侧 `产品总览`。

讲解要点：

- 产品不是功能拼盘，主线是“社群建店 -> 审核货盘 -> 分享成交 -> 履约售后 -> 收益结算”。
- 三场景均规划真实入口，但不等于开放全部供给。
- 首发优先证明交易和责任闭环，AI 与群接龙不能提前抢开发资源。

### 路径 2：社群建店

操作：选择 `社群建店`，或在总览手机页点击 `开始创建社群`。

讲解要点：

- 团长选择一个主类型与多个兴趣标签。
- 主类型决定主页模板和首屏排序，标签影响推荐货架。
- 系统不自动上架供给，团长仍需确认选品，供给发布仍需审核。

### 路径 3：社群店铺

操作：点击 `生成专属店铺主页`，或选择 `社群店铺`。

讲解要点：

- 店铺位于同一平台小程序中，不为每名团长创建独立小程序。
- 页面同店展示活动票务、虚拟权益商品和到店服务示例。
- 消费者可以识别供给发布方、履约方与推荐关系。

### 路径 4：审核货盘

操作：点击 `去选货`，或选择 `审核货盘`。

讲解要点：

- 供给库按场景分类并经过平台审核。
- 电商页展示物流、自提和虚拟权益三种不同交付类型。
- 食品生鲜、美容医疗、责任不清商品和上门服务不进入首发白名单。

### 路径 5：推广促销

操作：在虚拟权益商品点击 `选择`，或选择 `推广促销`。

讲解要点：

- 页面同时提示固定佣金与加价帮卖，两者互斥。
- 示例选择加价帮卖，展示底价、终端价、平台费、优惠券和推广收益。
- 推广团长发出的优惠由其收益承担，不侵蚀发布方底价；不开放下级推广。

### 路径 6：订单收益

操作：点击 `上架并生成分享卡`，或选择 `订单收益`。

讲解要点：

- 虚拟权益订单展示加价帮卖与权益已发放。
- 物流订单展示固定佣金与发布方退换责任。
- 到店服务取消订单展示收益和平台费按比例冲回。
- 自动结算仅为目标状态，需通过支付及退款回退核验。

### 路径 7：可行性看板

操作：选择 `可行性看板`。

讲解要点：

- 开发需统一支持票务、物流、自提、虚拟权益与到店预约五种履约状态。
- 运营需带回社群模板、审核货盘和白名单供给。
- 支付/合规需带回结算回退、优惠承担、平台规则与责任披露结论。
- 二期再评估 AI 营销话术和群接龙，不开放多级分销或高风险供给。

## 3. 页面验收表

| 页面 | 必须可见的业务表达 | 当前属性 |
| --- | --- | --- |
| 产品总览 | 三场景、五种履约、首发与二期边界 | 产品主线 |
| 社群建店 | 主类型、兴趣标签、模板匹配、不自动上架 | 首发底座 |
| 社群店铺 | 三类货架、履约责任披露 | 首发底座 |
| 审核货盘 | 活动/电商/服务分类，物流/自提/虚拟权益 | 首发底座 |
| 推广促销 | 固定佣金/加价互斥、优惠承担、一级限制 | 首发底座 |
| 订单收益 | 多履约订单、退款冲回、自动结算核验边界 | 待支付核验 |
| 可行性看板 | 开发、运营、支付/合规任务与后置能力 | 会议任务入口 |

## 4. 展示提醒

- 原型展示的是目标产品与预研任务，不表示真实小程序后端、自动分账或供给审核能力已经实现。
- 页面继续使用 Airbnb inspired 共享 tokens，不新增第二套视觉规范。
- 原型中的供给、价格、收益与优惠为演示数据，不作为经营承诺。

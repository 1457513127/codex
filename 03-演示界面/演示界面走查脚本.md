## 1. 演示定位

本原型用于产品负责人同步已确定的功能蓝图，并让开发与运营理解各自的前置任务。它展示目标产品流程和待核验能力，不表示真实后端、自动结算或三类业务均已上线。

会议中统一强调：

- 首轮蓝图覆盖 `活动`、`电商商品`、`付费内容/课程包`。
- 团长可发布供给，也可进行一次直接帮卖；推广方允许加价但不允许发展下级。
- 平台收入规划为经营工具订阅与按终端成交额计提的交易服务费。
- 自动结算需核验既有支付链路；本地服务、广告和直播训练营暂缓。

## 2. 主讲顺序

### 路径 1：产品总览与阶段路线

操作：选择左侧 `产品总览与路线`。

讲解要点：

- 三场景是产品版图，不是现场投票议题。
- 分期路线为蓝图与底座、模块预研、门槛后上线、后续扩展。
- 团队当前任务是预研和摸排，不能把目标能力描述成已经上线。

### 路径 2：团长店铺主页

操作：选择 `团长店铺主页`。

讲解要点：

- 经营型团长是店铺经营者，也是订阅付费主体。
- 同一店铺可承载自己的供给和一级帮卖供给，并按活动、商品、课程包组织。
- 消费者必须能区分发布/履约责任方、推广者和平台售后入口。

### 路径 3：发布供给

操作：选择 `发布供给`。

讲解要点：

- 三场景共用发布者、底价、审核、允许帮卖和售后责任字段。
- 商品示例明确由发布方发货与处理退换，推广团长不承担商品履约。
- 活动需要场次、名额、核销和退改；课程包需要授权入口与交付状态。

### 路径 4：一级帮卖与加价

操作：选择 `选品帮卖`，或在店铺页点击活动卡片的 `查看`。

讲解要点：

- 推广团长可以设置终端售价，示例展示供给底价、终端价、平台费和推广收益。
- 平台交易服务费按消费者支付的终端成交额计提。
- 页面明确“只支持一次直接帮卖”，不引入多级分销。

### 路径 5：订单与收益

操作：选择 `订单与收益`，或在帮卖页点击 `加入店铺并分享`。

讲解要点：

- 收益区同时表达订阅和成交抽佣两条收入线。
- 帮卖课程包订单用于展示发布方底价结算和推广收益。
- 商品部分退款用于展示平台费、发布结算与推广收益按比例回退。
- 自动结算只作为目标状态，支付复用核验未完成前不得承诺真实打款。

### 路径 6：运营与可行性看板

操作：选择 `可行性看板`。

讲解要点：

- 开发预研：对象模型、场景状态机、一级加价、权限、结算复用。
- 运营摸排：三类供给、经营团长需求、履约和售后真实成本。
- 支付/合规：双角色结算、退款冲回、商品责任、内容授权和订阅协议。
- 明确本轮不做本地服务、广告、直播训练营和多级分销。

## 3. 页面与功能规划对应表

| 演示页 | 表达的目标能力 | 是否已经可上线 |
| --- | --- | --- |
| 产品总览与路线 | 三场景版图与分期开放 | 规划同步 |
| 团长店铺主页 | 店铺、三类货架、责任披露、订阅身份 | 待预研 |
| 发布供给 | 场景字段、审核、底价、开放帮卖 | 待预研 |
| 选品帮卖 | 一级加价、终端额抽佣、归因 | 待预研 |
| 订单与收益 | 订阅、收益账本、按比例退款回退、自动结算状态 | 待支付核验 |
| 可行性看板 | 研发/运营/支付/合规前置清单 | 会议任务入口 |

## 4. 会后记录重点

| 团队 | 记录内容 |
| --- | --- |
| 开发 | 共用数据对象能否支撑三类场景、哪些模块需独立状态机、结算复用缺口 |
| 运营 | 三类供给和团长样本、底价与终端定价预期、履约/售后风险 |
| 支付/合规 | 自动结算、部分退款冲回、商品责任、内容授权、订阅协议的可执行边界 |
| 产品 | 将核验结论回写蓝图，区分已确认能力和待验证能力 |

## 5. 展示提醒

- 原型中的店铺、金额、供给和收益为讲解示意，不代表已产生真实经营数据。
- 页面继续使用 Airbnb inspired 共享 tokens，新增场景不改变视觉基线。
- 三场景同步规划不等于全部同时真实上线；是否开放以可行性核验结果为准。

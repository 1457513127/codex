## 1. 当前一句话定位

> 和趣是面向私域社群经营者的社群变现平台，服务已有社群经营者和零基础新手群主，把微信群、兴趣圈层和本地社区里的信任关系转化为可发布、可成交、可履约、可复购的社群生意。

当前核心价值表达：和趣不是单一商品卖货工具，也不是单纯帮团长分销的平台，而是用“社群经营者 + 可信供给 + 消费者订单 + 平台运营监督”的方式，把群聊里的口头信任交易变成有规则、有凭证、有售后和有收益解释的交易闭环。

当前执行基线以 [和趣私域社群变现平台产品蓝图](./01-研究与规划/三场景私域变现平台产品蓝图.md) 为准。

## 2. 当前产品主线

`选择经营方向 -> 社群冷启动 -> 建店 -> 商品库/供给库 -> 开团发布 -> 分享成交 -> 履约售后 -> 收益结算 -> 复购经营 -> 选品中心放大`

当前变现场景优先级：

- P0 首发主验证：活动变现、卡券变现。活动用于验证社群信任到报名/核销/售后的闭环，卡券用于验证标准化凭证、核销和复购。
- P1 首发对照/受控验证：低风险电商变现。本阶段只做标准商品和自提商品，不把复杂供应链或强售后品类作为主线。
- P1/P2 受控后置：本地服务。V1 可保留到店预约/核销的受控能力，规模化服务供给、上门服务和复杂履约后置。
- P2/P3 暂不独立做：内容/课程。只允许以体验课、工作坊、活动或卡券供给出现，不在 V1 建独立内容付费、系统课交付或讲师分账体系。

核心口径：

- 新手群主是核心客户之一，首发“起号”只定义为社群冷启动。
- 已有社群经营者侧重从 1 到 N：提升发布效率、成交统计、履约售后和收益解释。
- 零基础新手群主侧重从 0 到 1：明确服务谁、如何承接社群、如何完成第一笔真实交易。
- 供给方负责供给真实性、交付能力和基础履约承诺；平台运营负责准入审核、风险暂停、纠纷协同和账本监督，不在支付链路核验前承诺真实自动打款。
- 消费者/团员通过社群关系或内容入口下单，但需要平台化购买须知、核销/履约凭证和售后入口。
- 双通道与多端只作为产品承载方式，不作为项目主题。
- V1 是同一微信小程序 C/B 双通道，不拆两个小程序。
- 后续团长 App 是 B 端专业经营工具，微信、小红书、抖音小程序是 C 端消费入口。
- 所有端共用商品库、开团、订单、售后、归因和收益账本。

## 3. 当前范围

### 当前应做

- 建设社群建店、商品库/供给库和开团发布台。
- 增加新手社群冷启动基础：社群类型选择、新手店铺模板、首单开团清单和任务路径。
- 首期重点验证活动报名、卡券团购和低风险普通团购，其中活动和卡券优先级最高。
- 扩充活动货盘白名单，优先覆盖亲子家庭、青年兴趣、创业职场和社区生活。
- 建设订单中心、售后服务台和收益账本。
- 保留一级固定佣金分享赚和一级加价帮卖。

### 当前不做

- 多级分销、发展下级、层层转单。
- 保证金账户、经销商层级、会员分层可见。
- 食品生鲜、医疗美容、上门服务、高风险户外、多日复杂行程、强效果承诺活动和独立内容/课程付费体系。
- 自动选货、自动上架、自动外部发布营销内容。
- V1 同时建设小红书/抖音小程序消费端或团长 App。
- 按平台拆分多套商品库、多套订单、多套售后和多套结算。
- 支付链路未核验前的真实自动打款承诺。

## 4. 活动变现货盘口径

| 层级 | 活动方向 | 示例 |
| --- | --- | --- |
| 首发白名单 | 亲子家庭 | 亲子手作、研学体验、儿童剧/展览、亲子运动、营地轻体验 |
| 首发白名单 | 青年兴趣 | Citywalk、桌游社交、摄影采风、手作体验、音乐/观影活动、轻运动体验 |
| 首发白名单 | 创业职场 | 创业沙龙、资源对接、项目路演、行业交流、技能工作坊 |
| 首发白名单 | 社区生活 | 邻里集市、社区体验课、便民活动、节日主题活动、社区亲子日 |
| 受控扩展 | 文旅休闲 | 周边短途、非遗体验、博物馆/展馆讲解、城市文化路线 |
| 受控扩展 | 健康运动 | 瑜伽、羽毛球、骑行、徒步轻路线、运动体验课 |
| 受控扩展 | 学习成长 | 读书会、公开课、技能体验、亲子教育讲座 |

暂不开放：高风险户外、多日复杂行程、强效果承诺培训、医疗健康承诺、金融投资承诺、未成年人高风险托管类活动。

## 5. 市场与证据口径

市场空间采用 TAM / SAM / SOM：

- TAM：私域电商和本地生活交易大盘。
- SAM：微信小程序、社群团购、卡券核销、本地活动/服务交易。
- SOM：首期可触达城市、已有团长、新手群主、供给和真实订单。

外部数据只用于说明方向值得验证，不证明和趣已拥有收入或市场份额。
